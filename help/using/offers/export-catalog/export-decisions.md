# 데이터 집합 {#decisions-dataset} 결정

오퍼가 수정될 때마다 결정을 위해 자동으로 생성된 데이터 세트가 업데이트됩니다.

![](../assets/dataset-activities.png)

데이터 세트에서 가장 최근에 성공한 일괄 처리가 오른쪽에 표시됩니다. 데이터 세트에 대한 스키마의 계층 구조 보기가 왼쪽 창에 표시됩니다.

>[!NOTE]
>
>[이 섹션](../export-catalog/access-dataset.md)에서 오퍼 라이브러리의 각 개체에 대해 내보낸 데이터 집합에 액세스하는 방법을 알아봅니다.

의사 결정(이전의 오퍼 결정)은 의사 결정 프로세스를 제어하는 데 사용됩니다. 주제/카테고리별로 오퍼를 좁히기 위해 전체 인벤토리에 적용된 필터를 지정하고, 통합 제약 조건으로 인해 사용 가능한 모든 개인화 오퍼의 자격이 박탈되는 경우에 대비책 옵션을 지정하여 해당 오퍼를 기술적으로 예약된 공간에 맞는 오퍼로 인벤토리의 범위를 좁히도록 배치를 지정합니다.

다음은 **[!UICONTROL Decision Object Repository - Decisions]** 데이터 집합(이전의 결정 개체 저장소 - 활동이라고 함)에 사용할 수 있는 모든 필드의 목록입니다.

## _경험

### 의사 결정

#### 기준

각 결정 기준 세트에 제약 조건 세트가 포함되어 있는 결정 기준을 정의합니다.

유형:array

* **설명**

   기준 설명입니다. 이 기준은 어떻게, 왜 이 기준이 만들어졌는지, 그리고 그것이 결정에 어떤 영향을 주는지에 대한 사람이 읽을 수 있는 의도를 전달하는 데 사용됩니다.

   유형: 문자열

* **옵션 선택**

   옵션 선택 사항은 이 컨텍스트에서 옵션의 유효성/적용 가능성을 정의합니다.

   유형:개체

   * **설명**

      옵션 선택 설명. 이 옵션은 이 옵션 선택이 어떻게 구성되었는지 및/또는 어떤 옵션이 일치하는지 사람이 읽을 수 있는 의도를 전달하는 데 사용됩니다.

      유형: 문자열

   * **옵션 필터**

      첨부된 태그를 사용하여 인벤토리의 옵션과 일치하는 태그 기반 필터에 대한 참조입니다. 값은 참조되는 결정 규칙의 URI(@id)입니다. 스키마 https://ns.adobe.com/experience/decisioning/filter을 참조하십시오.

      유형: 문자열

   * **프로파일 제한 유형**

      제약 조건이 현재 설정되어 있는지 여부 및 제약 조건이 어떻게 표현되는지 결정합니다. 필터 쿼리 또는 하나 이상의 세그먼트 멤버십을 통해 만들 수 있습니다.

      유형: 문자열

   * **옵션 목록**

      필터 쿼리를 평가하지 않고 옵션을 직접 지정하는 목록입니다. 옵션 목록 또는 옵션 필터 규칙을 지정할 수 있습니다.

      유형:array

      <!--Missing title under Option List? Desc = An identifier of an decision option entity. The value value refers to an `@id` property of a decision option.-->

* **배치**

   배치 제약 조건에서는 이 기준이 나열된 배치에만 적용됨을 나타냅니다. 대상 배치가 `xdm:placements` 목록에 있는 경우에만 고려되는 옵션 선택 사항입니다. 그렇지 않으면 전체 결정 기준을 건너뜁니다. &#39;xdm:placement&#39; 목록이 생략되거나 비어 있으면 대상 배치에 대한 기준이 고려됩니다. 여기에 나열된 배치는 옵션 선택에 대한 암시적 기준을 적용합니다. 고려되는 옵션에는 대상 배치를 위한 표현이 있어야 합니다.

   유형:array

   * **배치 식별자**

   배치 엔티티에 대한 참조입니다. 값은 참조되는 위치의 URI(@id)입니다. 스키마 https://ns.adobe.com/experience/decisioning/placement을 참조하십시오.

   유형: 문자열

* **프로필 제한**

   현재 이 컨텍스트에서 프로필 제약 조건은 옵션 선택이 이 프로필 ID에 적합한지를 결정합니다. 프로파일 제약 조건이 각 옵션의 값을 고려할 필요가 없는 경우(예: 옵션 선택에서 옵션이 불변인 경우, &#39;false&#39;로 평가되는 프로파일 제약 조건은 전체 옵션 선택을 취소합니다.) 반면에 매개 변수로 옵션을 선택하는 프로파일 제한 규칙은 옵션 선택의 각 자격 옵션에 대해 평가됩니다.

   유형:개체

   * **설명**

      프로필 제한 설명입니다. 이 프로필 제약 조건이 어떻게 구성되었는지 및/또는 어떤 옵션이 포함되거나 제외될 것인지에 대해 읽을 수 있는 의도를 전달하는 데 사용됩니다.

      유형: 문자열

   * **자격 조건 규칙**

      지정된 프로필 및/또는 기타 상황에 맞는 XDM 개체에 대해 true 또는 false로 평가되는 결정 규칙에 대한 참조입니다. 이 규칙은 해당 옵션이 지정된 프로필에 적합한지를 결정하는 데 사용됩니다. 값은 참조되는 결정 규칙의 URI(@id)입니다. 스키마 https://ns.adobe.com/experience/decisioning/rule을 참조하십시오.

      유형: 문자열

   * **프로파일 제한 유형**

      제약 조건이 현재 설정되어 있는지 여부 및 제약 조건이 어떻게 표현되는지 결정합니다. 규칙 또는 하나 이상의 세그먼트 멤버십을 통할 수 있습니다.

      유형: 문자열

      가능한 값:&quot;none&quot;, &quot;eligibilityRule&quot;, &quot;anySegments&quot;, &quot;allSegments&quot;, &quot;rules&quot;

      기본값:&quot;none&quot;

   * **세그먼트 식별자**

      세그먼트의 식별자

      문자열:array

      * **식별자**

         관련 네임스페이스에 있는 세그먼트의 ID입니다.

         유형: 문자열

      * **네임스페이스**

         `xid` 속성과 연결된 네임스페이스입니다.

         유형:개체

         * **코드**

            코드는 네임스페이스에 대해 읽을 수 있는 식별자이며 ID 그래프 처리에 사용되는 기술 네임스페이스 id를 요청하는 데 사용할 수 있습니다.

            유형: 문자열
      * **경험 식별자**

         이 값이 있는 경우 이 값은 모든 네임스페이스의 모든 네임스페이스 범위 식별자에 대해 고유한 네임스페이스 간 식별자를 나타냅니다.

         유형: 문자열


* **등급 세부 사항**

   등급(우선 순위). 의사 결정 기준의 컨텍스트를 고려하여 \&quot;best option\&quot;을 선택하는 방법을 정의합니다. 프로필 제약 조건을 충족하는 선택한 모든 옵션 중에서 순위를 선택하여 제안 대상 상위(또는 상위 N) 옵션을 결정합니다.

   유형:개체

   * **주문 평가**

      하나 이상의 결정 옵션에 대한 상대적 순서 평가. 서수 값이 낮은 옵션에는 더 높은 옵션이 선택됩니다. 이 방법으로 결정된 값은 순서를 지정할 수 있지만 그 사이의 거리는 측정할 수 없으며 합이나 제품을 계산할 수 없습니다. 중간값과 모드는 서수 데이터에 사용할 수 있는 중앙 경향의 유일한 측정값입니다.

      유형:개체

      * **채점 함수**

         이 결정 옵션에 대한 숫자 점수를 계산하는 함수에 대한 참조입니다. 그러면 결정 옵션이 해당 점수에 따라 순서(등급)가 지정됩니다. 이 속성의 값은 옵션 시 호출할 함수의 URI(@id)입니다. 스키마 https://ns.adobe.com/experience/decisioning/function을 참조하십시오.

         유형: 문자열

      * **주문 평가 유형**

         사용할 주문 평가 메커니즘, 결정 옵션의 정적 우선 순위, 모든 옵션에 대한 숫자 값을 계산하는 점수 함수 또는 순서를 지정할 목록을 받는 등급 전략을 지정합니다.

         유형: 문자열

      * **순위 전략**

         결정 옵션 목록의 순위를 매기는 전략에 대한 참조입니다. 결정 옵션이 순서대로 표시됩니다. 이 속성의 값은 옵션 시 호출할 함수의 URI(@id)입니다. 스키마 https://ns.adobe.com/experience/decisioning/rankingStrategy을 참조하십시오.

         유형: 문자열
   * **우선순위**

      다른 모든 옵션에 상대적인 단일 결정 옵션의 우선 순위입니다. 순서 함수를 지정하지 않은 옵션은 이 속성을 사용하여 우선 순위가 지정됩니다. 우선 순위가 높은 옵션은 낮은 우선 순위 옵션 전에 선택됩니다. 두 개 이상의 적격 옵션이 가장 높은 우선 순위 값을 공유하는 경우 하나는 균일한 임의 상태로 선택되고 결정 제안에 사용됩니다.

      유형:정수

      최소 값:0

      기본값:0

유형: 문자열

유형: 문자열
