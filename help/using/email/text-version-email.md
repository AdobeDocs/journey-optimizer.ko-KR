# 이메일의 텍스트 버전 관리 {#text-version-email}

이메일 본문의 텍스트 버전을 만들어 두면 HTML 콘텐츠를 표시할 수 없을 때 사용할 수 있습니다.

보안 관점에서 볼 때, HTML 이메일은 서식 및 링크에 의존하는 악성 스크립트, 추적 픽셀 또는 피싱 시도와 같은 위험을 초래할 수 있으므로 일반 텍스트 버전을 제공하는 것이 중요합니다. 일반 텍스트는 공격 표면을 감소시키며, 종종 보안을 중시하는 수신자나 HTML을 제한하거나 제거하는 회사 이메일 시스템에서 선호됩니다. 두 버전을 모두 제공하면 수신자가 보안 및 개인 정보 보호 요구 사항에 맞는 형식을 선택할 수 있습니다.

## 기본 텍스트 버전에 액세스 {#plain-text-default}

기본적으로 이메일 디자이너는 개인화 필드를 포함한 이메일의 **[!UICONTROL 일반 텍스트]** 버전을 생성합니다. 이 버전은 자동으로 생성되어 콘텐츠의 HTML 버전과 동기화됩니다.

기본 텍스트 버전에 액세스하려면 이메일 콘텐츠에서 **[!UICONTROL 일반 텍스트]** 아이콘을 선택하십시오.

![](assets/text_version_3.png)

## 사용자 정의 텍스트 버전 사용 {#plain-text-custom}

일반 텍스트 버전에 대해 다른 콘텐츠를 사용하려면 다음 단계를 따르십시오.

1. 이메일에서 **[!UICONTROL 일반 텍스트]** 아이콘을 선택합니다.

1. **[!UICONTROL HTML과 동기화]** 전환을 사용하여 동기화를 비활성화하십시오. 확인 표시를 클릭하여 선택을 확인합니다.

   ![](assets/text_version_2.png)

1. 그런 다음 사용자 정의 일반 텍스트 버전을 원하는 대로 편집할 수 있습니다.

>[!CAUTION]
>
> * 동기화를 사용하지 않도록 설정하면 **[!UICONTROL 일반 텍스트]** 보기에서 변경한 내용이 HTML 보기에 반영되지 않습니다.
>
> * 일반 텍스트 콘텐츠를 업데이트한 후 **[!UICONTROL HTML과 동기화]** 옵션을 다시 사용하도록 설정하면 변경 내용이 손실되고 HTML 버전에서 생성된 텍스트 콘텐츠로 바뀝니다.

## 사용자 정의 일반 텍스트 버전을 사용해야 하는 경우 {#when-to-use}

사용자 정의 일반 텍스트 버전을 만드는 시기와 자동 동기화를 사용하는 시기를 이해하면 최적의 이메일 전달 및 가독성을 보장할 수 있습니다.

### 다음과 같은 경우 사용자 정의 일반 텍스트 사용(동기화 비활성화):

* **복잡한 HTML 레이아웃** - HTML 이메일에 다중 열 레이아웃, 표 또는 일반 텍스트로 잘 번역되지 않는 복잡한 CSS가 포함되어 있습니다.
* **시각적으로 중요한 콘텐츠** - 이메일은 이미지에 많이 의존하며, 이미지가 비활성화된 클라이언트에 대해 설명 텍스트 대체 요소를 제공하려고 합니다.
* **다른 메시징 구조** - 일반 텍스트 판독기에 최적화된 단순화되거나 재구성된 메시지 구조를 제공하려 합니다.
* **접근성 요구 사항** - 접근성 표준을 충족하려면 특정 일반 텍스트 서식이 필요합니다.
* **기존 전자 메일 클라이언트** - 대상에는 특별히 형식이 지정된 콘텐츠가 필요한 이전 전자 메일 클라이언트(예: Outlook 2003, 텍스트 전용 모바일 클라이언트)의 사용자가 포함됩니다.
* **ASCII 서식** - ASCII 아트, 표 또는 특정 줄 바꿈과 같은 특정 일반 텍스트 서식을 포함합니다.

### 다음과 같은 경우 자동 동기화(기본값) 사용:

* **간단한 HTML 디자인** - HTML 이메일은 일반 텍스트로 잘 변환되는 간단하고 선형적인 구조를 가지고 있습니다.
* **컨텐츠 일관성** - HTML과 일반 텍스트 버전 간의 정확한 일관성을 유지하려 합니다.
* **자주 업데이트** - 전자 메일 콘텐츠를 정기적으로 업데이트하므로 수동으로 중복되지 않도록 해야 합니다.
* **Personalization이 제대로 작동합니다.** - 개인화 필드가 두 형식 모두에서 제대로 작동합니다.
* **시간 제한** - 일반 텍스트 추가 사용자 지정 없이 전자 메일을 빠르게 시작해야 합니다.

## 실제 예 {#practical-examples}

다음 예제에서는 사용자 정의 일반 텍스트 또는 자동 동기화 사용 여부를 결정하는 데 도움이 되는 실제 시나리오를 보여 줍니다. 각 예제에서는 컨텍스트, 권장 접근 방식 및 결정 배경에 대한 이유를 설명합니다.

+++예제 1: 복잡한 레이아웃의 마케팅 뉴스레터

**시나리오:** 이미지, 스타일이 지정된 단추 및 색상으로 구분된 섹션이 포함된 다중 열 뉴스레터입니다.

![](assets/text_version_ex1.png)

**권장 사항:** 사용자 지정 일반 텍스트를 사용합니다(동기화 비활성화).

**사용자 지정 일반 텍스트인 이유:** HTML 버전에서는 배너 이미지, 스타일이 지정된 단추 및 색상으로 구분된 섹션이 포함된 3열 그리드 레이아웃을 사용합니다. 이러한 시각적 요소는 자동 동기화를 통해 일반 텍스트로 잘 번역되지 않아 콘텐츠가 복잡하고 읽기 어렵습니다. 사용자 정의 일반 텍스트 버전을 사용하면 내용을 선명한 섹션 헤더와 적절한 형식의 링크를 사용하여 스캔하기 쉬운 선형 형식으로 재구성할 수 있습니다.

**사용자 지정 일반 텍스트 예:**

```
================================================
YOUR BRAND - MONTHLY NEWSLETTER
December 2025
================================================

🌟 FEATURED ARTICLE
"10 Ways to Optimize Your Customer Journeys"
Read more: https://example.com/articles/optimize-journeys

📢 UPCOMING WEBINAR
"Mastering Email Personalization"
December 15, 2025 at 2:00 PM EST
Register: https://example.com/webinar/register

📦 NEW PRODUCTS
- Winter Collection: https://example.com/winter
- Holiday Gift Guide: https://example.com/gifts

================================================
Website: https://example.com
Unsubscribe: https://example.com/unsubscribe
================================================
```

**시나리오:** 정형 데이터(주문 번호, 품목, 가격, 배송 세부 사항)가 있는 주문 확인.

![](assets/text_version_ex2.png)

**권장 사항:** 자동 동기화를 사용합니다.

**자동 동기화가 작동하는 이유:** 주문 확인은 HTML에서 일반 텍스트로 자연스럽게 변환되는 간단하고 선형의 구조를 가지고 있습니다. 정보는 논리적으로(주문 세부 사항 → 품목 → 합계 → 배송) 흐르고 주문 번호 및 고객 이름과 같은 개인화 필드는 두 형식 모두에서 동일하게 작동합니다. 구조화된 테이블 형식의 데이터는 수동으로 조정할 필요 없이 깔끔하게 변환되므로 시간을 절약하면서 명확성을 유지합니다.

## 일반적인 사용 사례 {#common-use-cases}

다음 사용 사례에서는 사용자 지정 일반 텍스트 버전을 만드는 것이 유용한 상황을 보여 줍니다(동기화 비활성화). 각 예제는 HTML 버전과 사용자 정의 일반 텍스트 솔루션이 이를 해결하는 방법에 대한 과제를 보여줍니다.

+++사용 사례 1: 제품 카탈로그 이메일

**과제:** HTML은 이미지, 가격 및 구매 단추가 있는 제품 표를 표시합니다.

**일반 텍스트 솔루션:** 제품 이름, 가격 및 직접 링크가 명확한 구조화된 목록을 만듭니다.

```
FEATURED PRODUCTS
=================

1. Premium Leather Wallet
   Price: $89.99
   View product: https://example.com/product/wallet
   
2. Designer Sunglasses
   Price: $129.99
   View product: https://example.com/product/sunglasses
```

+++

+++사용 사례 2: 환영 이메일 시리즈

**과제:** 회사 로고 및 스타일이 지정된 브랜드 환영 전자 메일

**일반 텍스트 솔루션:** ASCII Art 또는 텍스트 서식을 사용하여 시각적 계층 구조를 만듭니다.

```
***************************************************
*                                                 *
*     WELCOME TO [BRAND NAME]                    *
*     We're thrilled to have you!                *
*                                                 *
***************************************************
```

+++

+++사용 사례 3: 설문 조사 또는 피드백 요청

**과제:** HTML에 스타일이 지정된 단추와 양식 요소가 포함되어 있습니다.

**일반 텍스트 솔루션:** 지침과 함께 일반 텍스트 링크를 제공합니다.

```
We'd love your feedback!
------------------------

Please take 2 minutes to complete our survey:
https://example.com/survey/customer-feedback

Your input helps us improve our service.
```

## 자주 묻는 질문 {#faq}

**개인 설정 필드가 일반 텍스트로 작동합니까?**\
예. `{{profile.firstName}}`과(와) 같은 개인화 필드는 HTML 및 일반 텍스트 버전 모두에서 동일하게 작동합니다.

**일반 텍스트 버전을 테스트하려면 어떻게 해야 합니까?**
* 이메일 Designer에서 **[!UICONTROL 일반 텍스트]** 보기로 전환합니다. [방법 알아보기](#text-version-email)
* 테스트 이메일을 이전 버전의 Pine 또는 기본 모바일 이메일 앱과 같은 텍스트 전용 이메일 클라이언트에 보냅니다.

**일반 텍스트 버전을 만드는 것을 잊어버리면 어떻게 됩니까?**\
시스템에서는 HTML에서 일반 텍스트 버전을 자동으로 생성합니다. 이 버전은 형식이 최적이 아닐 수 있지만 텍스트 전용 클라이언트에게 전달됩니다.

**HTML과 일반 텍스트에서 다른 개인화를 사용할 수 있습니까?**\
예. 동기화를 비활성화하면 다른 개인화 필드 또는 콘텐츠를 사용하여 각 버전을 독립적으로 사용자 지정할 수 있습니다.

**일반 텍스트만 지원하는 전자 메일 클라이언트는 무엇입니까?**\
일부 최신 클라이언트는 텍스트 전용이지만 일부 기업 이메일 정책, 접근성 도구 및 이전 모바일 장치에는 일반 텍스트가 표시될 수 있습니다. 또한 HTML 렌더링이 실패할 때 대비됩니다.

**일반 텍스트 버전을 얼마나 자주 업데이트해야 합니까?**\
HTML 콘텐츠를 크게 변경할 때마다 업데이트합니다. 핵심 메시지가 동일하게 유지되는 경우 부 HTML 수정에는 일반 텍스트 업데이트가 필요하지 않을 수 있습니다.

**일반 텍스트 전자 메일에 링크를 포함할 수 있습니까?**\
예! 전체 URL(예: https://example.com/page)을 포함하면 대부분의 이메일 클라이언트가 자동으로 클릭할 수 있습니다.

**이미지를 일반 텍스트로 포함해야 합니까?**\
아니요. 일반 텍스트는 이미지를 지원하지 않습니다. 대신 이미지에 표시되는 내용을 설명하거나 온라인으로 볼 수 있는 링크를 제공합니다.
