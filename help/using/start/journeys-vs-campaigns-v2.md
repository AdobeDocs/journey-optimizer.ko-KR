# 여정과 캠페인: 적합한 접근 방식 선택 {#journeys-vs-campaigns}

>[!BEGINSHADEBOX]

**이 페이지에서:** Adobe Journey Optimizer의 각 마케팅 사용 사례에 적합한 접근 방식을 선택할 수 있도록 여정을 작업 및 API 트리거 캠페인과 비교합니다. 오케스트레이션된 캠페인의 경우 [오케스트레이션된 캠페인 시작](../orchestrated/gs-orchestrated-campaigns.md)을 참조하세요.

>[!ENDSHADEBOX]

[!DNL Adobe Journey Optimizer]은(는) 고객에게 연락하여 참여할 수 있는 두 가지 주요 방법인 **여정** 및 **캠페인**&#x200B;을 제공합니다. 여정은 고객 행동을 기반으로 하는 실시간, 여러 단계 오케스트레이션을 위해 설계된 반면, 캠페인은 정의된 대상에게 일회성 또는 예약된 브로드캐스트를 보내거나 지연 시간이 짧은 개인화를 위해 edge로 인바운드 채널을 활성화하는 데 더 적합합니다.

>[!NOTE]
>
>**오케스트레이션된 캠페인**&#x200B;에는 전용 지침이 필요한 고유한 아키텍처 특성(허브측 배치 실행, 다중 엔터티 관계형 데이터)이 있습니다. 이들은 지나친 단순화를 방지하기 위해 아래 비교에 포함되지 않습니다. [오케스트레이션된 캠페인에 대해 자세히 알아보기](../orchestrated/gs-orchestrated-campaigns.md)

## 어떤 방법을 사용해야 합니까? {#decision-guide}

답변은 보통 한 가지 질문으로 이어집니다. *어떤 일이 발생해야 하며 누구에게 해야 합니까?*

**각 고객이 원하는 속도로 이동해야 하는 경우**(고객이 수행하는 작업에 따라 메시지를 받고, 기다리고, 다음 작업에 반응하는 경우) **여정**&#x200B;을(를) 사용합니다. 여정은 각 프로필의 위치와 수행한 작업을 추적하므로 온보딩 시퀀스, 장바구니 포기 흐름 또는 충성도 프로그램과 같은 여러 단계 경험에 이상적입니다.

**일정에 따라 여러 사람에게 메시지를 보내려면**(뉴스레터, 제품 발표, 시즌 홍보) **액션 캠페인**&#x200B;을 사용하세요. 대상에 있는 모든 사람이 프로필별 논리가 필요 없이 동시에 메시지를 수신합니다. 작업 캠페인은 짧은 지연 에지 개인화를 위한 인바운드 채널 활성화(인앱, 웹, 콘텐츠 카드, 코드 기반)도 지원합니다.

**외부 시스템에서 메시지를 즉시 트리거해야 하는 경우** — 전자 상거래 플랫폼의 주문 확인, 물류 시스템의 배송 경고, 앱에서 암호 재설정 — 단일 온디맨드 메시지에 대해 **API 트리거 캠페인**&#x200B;을 사용하거나, 트리거가 여러 단계 흐름을 시작해야 하는 경우 **단일 이벤트 여정**&#x200B;을 사용합니다.

고급 세분화, 다중 엔티티 데이터 또는 정확한 사전 전송 카운트가 포함된 **복잡한 일괄 처리 워크플로우가 필요한 경우**&#x200B;오케스트레이션된 캠페인 시작[&#128279;](../orchestrated/gs-orchestrated-campaigns.md)을 참조하세요.

>[!TIP]
>
>**어디서부터 시작할지 알 수 없습니까?** 대부분의 팀은 예약된 커뮤니케이션을 위해 행동, 이벤트 기반 경험 및 작업 캠페인에 여정을 사용합니다. 이 두 가지가 대부분의 사용 사례를 다룹니다.

## 비교 방법 {#quick-overview}

| 접근 방식 | 다음에 최적 | 실행 스타일 |
|----------|----------|-----------------|
| **여정** | 조건부 논리를 사용한 다단계 실시간 고객 경험 | 1:1 오케스트레이션 — 각 프로필의 속도에 맞게 조정 |
| **액션 캠페인** | 대상에 대한 예약 또는 반복 활성화 | 일괄 처리 실행 — 전송 시 함께 처리된 대상자 |
| **API 트리거 캠페인** | 외부 시스템의 이벤트 기반 또는 트랜잭션 메시지 | 온디맨드 실행 — 페이로드가 있는 API 호출에 의해 트리거됨 |

## 각 접근 방식의 작동 방식 {#key-distinctions}

### 여정: 1:1 실시간 오케스트레이션

여정은 각 프로필이 각자의 속도로 고유한 경로를 이동하는 캔버스입니다. AJO은 각 사람이 흐름 내에 있는 위치를 추적하고 수행하는 작업, 활동이 없는 기간 또는 프로필이 변경되었는지 여부에 관계없이 각 사람의 행동에 실시간으로 반응합니다.

주요 기능에는 단계 간에 개인화된 타이밍을 만드는 대기 활동, 프로필을 다른 경로로 라우팅하는 조건부 분기, 고객이 메시지를 받는 빈도를 제어하는 빈도 제한, 라이브로 전환하기 전에 논리를 확인하는 테스트 모드 등이 있습니다. 여정은 또한 여러 경로에서 A/B 실험을 위해 프로필을 백분율 기반 그룹으로 분할할 수 있습니다.

장바구니 포기 여정은 차이점을 명확하게 보여 줍니다.

```
Customer A: Abandoned cart → Wait 2 hours → No purchase? → Send reminder → Purchased? → End
Customer B: Abandoned cart → Wait 2 hours → Already purchased → End immediately
```

각 고객은 실제로 수행하는 작업에 따라 다른 타임라인을 경험합니다. [여정에 대해 자세히 알아보기](../building-journeys/journey.md)

### 캠페인: 일괄 처리 또는 트리거된 게재

캠페인은 한 번에 대상자의 모든 사람에게 또는 외부 시스템에서 호출한 경우 요청 시 단일 작업을 실행합니다. 캔버스 및 프로필별 상태 없음: 모든 프로필은 동일하게 처리됩니다.

**작업 캠페인** 예약된 대상(1회 또는 반복)에게 전달하고 다중 표면 인바운드 배달도 지원합니다. 대상 멤버십 또는 프로필 특성에 따라 메시지 변형을 만드는 타깃팅 규칙을 사용하여 캠페인당 최대 10개의 인바운드 채널 작업을 지원합니다.

**API 트리거된 캠페인**&#x200B;은(는) 외부 시스템이 API를 호출할 때 해당 호출에서 전송된 페이로드 데이터에 따라 메시지 개인화를 사용하여 즉시 실행됩니다.

```
Monday 9 AM → Send newsletter to 100,000 subscribers → All receive simultaneously
```

[캠페인에 대해 자세히 알아보기](../campaigns/get-started-with-campaigns.md)

| 미리 알림 시퀀스가 있는 장바구니 포기 | 여정 | 실시간 트리거, 대기 시간, 조건부 후속 조치 |
| 비헤이비어에 따라 비활성 사용자 재참여 | 여정 | 대상 자격, 개인화된 경로에 의해 트리거됨 |

| 주문 확인 또는 배송 알림 | API로 트리거되는 캠페인 | 외부 시스템 트리거, 즉각적인 일회성 게재 |
| API 트리거된 여러 단계 흐름 | 여정(단일 이벤트) | 외부 시스템은 API를 통해 이벤트를 전송하고, 여정은 후속 단계를 조정합니다 |
| 다중 엔티티 데이터가 포함된 복잡한 일괄 처리 워크플로 | 오케스트레이션된 캠페인 | [오케스트레이션된 캠페인 시작](../orchestrated/gs-orchestrated-campaigns.md)을 참조하세요. |

## 일반적인 질문 {#common-questions}

+++ 마케팅 전략에 여정과 캠페인을 결합할 수 있습니까?

예. 많은 조직은 다양한 시나리오에 대해 모든 접근 방식을 사용합니다.

* 동작, 실시간 참여를 위한 **여정**
* 예약된 통신 또는 인바운드 활성화를 위한 **작업 캠페인**
* 트랜잭션 메시지용 **API 트리거 캠페인**
* 복잡한 데이터 집약적 일괄 처리 캠페인에 대한 **오케스트레이션된 캠페인** — [오케스트레이션된 캠페인 시작](../orchestrated/gs-orchestrated-campaigns.md) 참조

모든 경우에 한 가지 접근 방식을 강요하지 않고 각 사용 사례에 적합한 도구를 사용하십시오.

+++

+++ 캠페인을 여정으로 전환하거나 그 반대로 전환할 수 있습니까?

아니요. 적절한 형식으로 경험을 다시 빌드해야 합니다. 하지만 콘텐츠, 대상 및 논리 개념을 재사용할 수 있습니다.

+++

+++ 어떤 접근 방식을 구축하는 것이 가장 쉽습니까?

작업 캠페인은 일반적으로 가장 간단한(대상자에게 제공되는 단일 터치포인트), API 트리거 캠페인, 여정 순으로 진행되며, 여러 단계 논리로 인해 더 많은 디자인 작업이 필요합니다.

+++

+++ 대규모 대상자에게 어떤 확장이 더 적합합니까?

세 가지 모두 확장성이 뛰어납니다. 올바른 선택은 패턴에 따라 다릅니다.

* **대상자 여정 읽기** 및 **작업 캠페인**&#x200B;은(는) 일괄 처리 대상자에 맞게 최적화되었습니다.
* **단일(이벤트 기반) 여정**&#x200B;은 이벤트가 발생할 때 개별적으로 프로필을 처리하므로 이벤트 볼륨 및 처리량에 따라 크기가 달라집니다.

대용량 데이터 세트 및 다중 엔터티 데이터를 사용하는 복잡한 세분화에 대해서는 [오케스트레이션된 캠페인](../orchestrated/gs-orchestrated-campaigns.md)을 참조하세요.

+++

+++ 여정 및 캠페인에서 동일한 대상을 사용할 수 있습니까?

예. [!DNL Adobe Experience Platform]에서 만든 대상은 여정, 작업 캠페인 및 오케스트레이션된 캠페인에서 사용할 수 있습니다. API 트리거 캠페인은 페이로드 기반으로 하며 사전 빌드된 대상을 동일한 방식으로 사용하지 않습니다.

+++

**FAQ:**
* **Q: 언제 작업 캠페인 대신 여정을 사용해야 합니까?** — 고객이 여러 접점에서 실시간 조건부 논리를 사용하여 원하는 속도로 이동해야 할 때 여정을 사용합니다. 사전 정의된 대상에게 예약 또는 인바운드 전달을 위한 작업 캠페인을 사용합니다.
* **Q: 액션 캠페인이 인바운드 채널에 전달할 수 있습니까?** — 예. 작업 캠페인은 짧은 대기 시간 개인화를 위해 가장자리로 인바운드 채널 활성화(코드 기반 경험, 인앱, 콘텐츠 카드, 웹)를 지원하며, 캠페인당 최대 10개의 인바운드 작업과 메시지 변형에 대한 타깃팅 규칙을 제공합니다.
* **Q: 오케스트레이션된 캠페인과 액션 캠페인을 구별하는 것은 무엇입니까?** — 오케스트레이션된 캠페인은 다중 엔티티 관계형 데이터, 정확한 사전 전송 카운트, 임시 대상 구성 및 페더레이션 데이터 지원을 통해 허브측 배치 실행을 실행합니다. 작업 캠페인은 Experience Platform 대상에 상태를 저장하지 않는 단일 실행 게재입니다.
* **Q: 언제 API 트리거 캠페인과 단일 이벤트 여정을 사용해야 합니까?** — 외부 시스템이 페이로드 데이터로 바로 단일 메시지를 트리거해야 하는 경우 API 트리거 여정을 사용하고, API 전송 이벤트 후 여러 단계 오케스트레이션이 필요한 경우 단일 이벤트 캠페인을 사용합니다.
* **Q: 여정과 캠페인을 동일한 마케팅 전략에 결합할 수 있습니까?** — 예. 행동 실시간 참여, 예약된 브로드캐스트 또는 인바운드 활성화에 대한 작업 캠페인, 트랜잭션 메시지에 대한 API 트리거 캠페인 및 복잡한 일괄 처리 워크플로우에 대한 오케스트레이션된 캠페인에 여정을 사용합니다.
