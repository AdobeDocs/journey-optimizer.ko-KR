# 역할 및 책임

Adobe Journey Optimizer을 통해 브랜드는 고객 라이프사이클 전반에 걸쳐 연결되고 상황에 맞는 고객 여정을 제공할 수 있습니다. 이를 통해 팀에서 대규모로 상호 작용을 개인화하고 고객 기대치와 비즈니스 목표의 방향성을 맞출 수 있습니다. 이 설명서에서는 Journey Optimizer를 효과적으로 사용하는 데 필요한 주요 역할과 그 책임 및 시작 방법을 설명합니다.

**중요 정보:** Adobe Journey Optimizer는 구체적인 책임에 따라 각기 역할을 정의합니다. 조직의 구조에 따라 한 개인이 여러 역할 또는 모든 역할을 수행할 수 있습니다.

## 역할 기반 빠른 시작 안내서

구현을 단순화하기 위해 Adobe Journey Optimizer은 전문 지식을 기반으로 작업을 특정 역할로 구성합니다. 각 역할은 원활한 고객 경험을 제공하는 데 필요한 필수 작업에 중점을 둡니다.

| 역할 | 주요 책임 | 주요 기술 | 일반적인 작업 |
|-------------------|----------------------------------|--------------------------------|-----------------------------------------------|
| **관리자** | 환경 설정 및 액세스 관리 | 시스템 구성, 사용자 관리, 보안 | 샌드박스 구성, 권한 관리, 채널 구성 설정 |
| **데이터 엔지니어** | 데이터 기반 및 아키텍처 | 데이터 모델링, XDM 스키마, 데이터 품질 | 스키마 및 데이터 세트 만들기, 데이터 수집 구성, 데이터 라이프사이클 관리 |
| **Developer** | 기술 구현 및 통합 | 모바일/웹 SDK, API, 이벤트 기반 아키텍처 | SDK 통합, 이벤트 구현, 사용자 지정 작업 엔드포인트 빌드 |
| **마케터** | 고객 경험 디자인 및 실행 | 여정 디자인, 콘텐츠 제작, 데이터 분석 | 여정 구축, 개인화된 콘텐츠 제작, 캠페인 최적화 |

각 역할은 Adobe Journey Optimizer 구현의 특정 단계를 다루고 체계적이고 효율적인 배포 프로세스를 보장합니다.

## 구현 순서와 역할 종속성

성공적인 Journey Optimizer 구현은 일반적으로 이 순서를 따르며, 이는 역할 간의 종속성을 반영합니다.

1. **관리자**: 환경 설정\
   관리자는 샌드박스를 구성하고, 액세스 제어를 설정하고, 채널 구성을 준비하여 기반을 다집니다. 다른 팀이 작업할 수 있도록 하려면 먼저 이 작업을 수행해야 합니다.
   * 개발, 스테이징 및 프로덕션 샌드박스 구성
   * 역할, 권한 및 OLAC(객체 수준 액세스 제어) 설정
   * 채널 구성(이메일, SMS, 푸시, 인앱, 웹, 콘텐츠 카드)
   * 하위 도메인 위임 및 IP 풀 설정
   * 제외 목록 및 동의 정책 구성

2. **데이터 엔지니어**: 데이터 기반 구축\
   데이터 엔지니어는 개인화를 지원하는 데이터 인프라를 구축하여 고객 데이터가 시스템으로 그리고 시스템을 통해 이동하는 방식을 정의합니다.
   * 고객 식별을 위한 ID 네임스페이스 만들기
   * XDM 스키마 디자인(프로필, 경험 이벤트, 관계형)
   * 데이터 세트를 설정하고 실시간 고객 프로필에 대해 활성화
   * 데이터 수집 구성(일괄 처리 및 스트리밍)
   * 복잡한 계산을 위해 계산된 속성 만들기
   * 여정에 대한 이벤트 및 데이터 소스 구성

3. **개발자**: 기술 통합 구현\
   개발자는 SDK를 통합하고, 이벤트를 전송하고, API 끝점을 구축하여 애플리케이션을 Journey Optimizer에 연결합니다. 이러한 구현을 통해 여정은 트리거하고 실행할 수 있습니다.
   * 푸시 알림 설정과 모바일 SDK(iOS/Android) 통합
   * 웹 경험을 위한 웹 SDK 구현
   * 애플리케이션에서 이벤트를 전송하여 여정 트리거
   * 외부 시스템 통합을 위한 사용자 지정 작업 엔드포인트 작성
   * Adobe Experience Platform Assurance을 사용하여 구현 테스트

4. **마케터**: 고객 경험을 디자인하고 실행합니다.\
   마케터는 모든 기본 작업을 활용하여 모든 채널에서 여정을 구축하고, 콘텐츠를 만들고, 고객 경험을 최적화합니다.
   * 세분화, CSV 업로드 또는 대상 구성을 사용하여 대상 작성
   * AI Assistant 및 템플릿을 사용하여 개인화된 콘텐츠 디자인
   * 이벤트 및 대상 트리거를 사용하여 다중 채널 여정 만들기
   * 실행 전 승인 워크플로우로 테스트
   * 보고 통찰력을 기반으로 성능 모니터링 및 최적화

**참고:** 이 순서가 일반적이지만 일부 활동은 동시에 발생할 수 있습니다. 예를 들어 데이터 엔지니어가 스키마를 구성하는 동안 개발자가 앱 통합 작업을 수행할 수 있습니다.

## 역할별 시작

각 역할은 역할의 초점에 맞게 조정된 구체적인 작업으로 시작됩니다. 이 초기 단계를 완료하면 전체 구현 프로세스에 대한 온보딩 및 정렬을 보다 원활하게 수행할 수 있습니다.

### 마케터용 {#for-marketers}

모든 채널에서 개인화된 고객 경험을 만드는 데 주력하십시오.

사용할 **주요 기능:**

* 대상을 만들고 여러 메서드(세그먼트 정의, CSV 업로드, 대상 구성)를 사용하여 세그먼트를 만듭니다
* 텍스트 및 이미지 생성을 위해 AI Assistant를 사용하여 콘텐츠 디자인
* 드래그 앤 드롭 디자이너를 통해 다중 채널 고객 여정 구축
* 전송 시간 최적화 및 충돌 관리를 활용하여 참여 극대화
* 게시하기 전에 콘텐츠를 테스트하고 승인 워크플로 사용
* 통합 보고 대시보드를 사용하여 성능 모니터링

**시작:** 미리 작성된 템플릿을 사용하여 간단한 시작 여정 또는 포기한 장바구니 복구 캠페인을 만듭니다.

[마케팅 담당자로 →](path/marketer.md)

### 데이터 엔지니어용 {#for-data-engineers}

개인화된 경험을 지원하는 데이터 기반을 구축하십시오.

**주요 책임:**

* ID 네임스페이스 만들기 및 ID 해결 구성
* 프로필 및 이벤트 데이터에 대한 XDM 스키마 디자인(표준 및 관계형)
* 데이터 세트를 설정하고 실시간 고객 프로필에 대해 활성화
* 일괄 처리 및 스트리밍 데이터 수집을 위한 소스 커넥터 구성
* 세그먼테이션을 단순화하기 위한 계산된 속성 만들기
* 여정 실행을 위한 이벤트 및 데이터 소스 구성
* 데이터 품질, 거버넌스 및 라이프사이클 관리

**시작:** ID 네임스페이스를 설정하고 필요한 필드 그룹을 사용하여 첫 번째 프로필 스키마를 만듭니다.

[데이터 엔지니어 로 시작하기 →](path/data-engineer.md)

### 관리자용 {#for-administrators}

조직의 Journey Optimizer 환경을 설정 및 관리합니다.

**주요 책임:**

* 개발, 테스트 및 프로덕션용 샌드박스 제작 및 관리
* 기본 제공 또는 사용자 정의 역할을 사용하여 역할 및 권한 구성
* OLAC(객체 수준 액세스 제어)를 적용하여 리소스 보호
* 이메일, SMS, 푸시, 인앱, 웹 및 콘텐츠 카드에 대한 채널 구성 설정
* 이메일 게재 가능성을 위해 하위 도메인 위임 및 IP 풀 만들기
* 제외 목록 및 허용 목록 관리
* 동의 정책 및 데이터 거버넌스 구성(Healthcare/Privacy Shield 사용)

**다음으로 시작:** 샌드박스를 구성하고 기본 역할 및 권한을 설정한 다음 채널 구성에 대해 팀과 작업합니다.

[관리자로 시작하기 →](path/administrator.md)

### 개발자용 {#for-developers}

Journey Optimizer을 애플리케이션에 연결하는 기술 통합을 구현합니다.

**주요 책임:**

* Adobe Experience Platform Mobile SDK 통합(iOS/Android)
* 웹 경험 및 웹 푸시 알림용 웹 SDK 구현
* 푸시 알림 자격 증명 및 인증서 구성
* 애플리케이션에서 이벤트를 전송하여 여정 트리거
* Journey Optimizer이 사용자 지정 작업을 통해 호출하는 API 끝점 빌드
* 웹, 모바일 및 기타 표면에 대한 코드 기반 경험 구현
* Adobe Experience Platform Assurance을 사용하여 구현 테스트 및 디버그
* 프로그래밍 방식 액세스를 위해 Journey Optimizer API 작업

**다음으로 시작:** Mobile 또는 Web SDK을 통합한 다음 첫 번째 이벤트를 구현하여 여정을 트리거합니다.

[개발자로 시작하기 →](path/developer.md)

## 크로스 역할 Collaboration

Journey Optimizer 구현이 성공하려면 모든 역할에서 공동 작업이 필요합니다.

* **관리자** 샌드박스, 권한 및 채널 구성을 설정하여 다른 역할을 사용하도록 설정합니다.
* **데이터 엔지니어**&#x200B;는 개발자와 마케터가 기반으로 하는 데이터 기반을 제공합니다.
* **개발자** 마케터가 여정을 트리거하는 데 사용하는 기술 통합을 구현합니다.
* **마케터**&#x200B;가 모든 팀에 데이터 품질, 기능 요청 및 사용자 경험에 대한 피드백을 제공합니다

**모범 사례:** 다양한 분야의 사람들과 정기적으로 회의를 열어 우선 순위를 조정하고, 진행 상황을 공유하고, 팀 간의 차단기를 해결하십시오.
