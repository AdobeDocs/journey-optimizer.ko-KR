# 역할 및 책임

브랜드는 Adobe Journey Optimizer를 사용해 고객 여정 전반에 걸쳐 연결되고 상황에 맞는 개인화된 경험을 제공할 수 있습니다. 규모, 속도 및 유연성에 대한 종합적인 초점을 바탕으로 구축된 Journey Optimizer는 통합 애플리케이션에 다음과 같은 세 가지 주요 가치 요소를 결합합니다.

* Adobe의 실시간 고객 프로필을 기반으로 하는 **실시간 고객 인사이트 및 참여**
* 실시간 캠페인과 일괄 여정, 최신 메시지 디자이너를 위한 통합 캔버스를 통해 **최신 옴니채널 오케스트레이션**
* 의사 결정 관리 및 AI/ML 기능을 통해 **지능형 의사 결정 및 개인화**

Journey Optimizer는 다양한 마케팅 요구 사항을 충족하는 두 가지 오케스트레이션 접근 방식을 제공합니다.

* **여정**: 각 고객이 비헤이비어 또는 이벤트에 의해 트리거된 각자의 속도로 진행하는 실시간 일대일 참여에 가장 적합합니다.
* **오케스트레이션된 캠페인**: 계절별 프로모션, 제품 출시 및 계정 기반 커뮤니케이션에 이상적인, 일정에 따라 여러 단계 워크플로우를 통해 대상자가 함께 진행하는 일괄 일대다 캠페인에 적합합니다.

이 통합 경험을 사용하면 대상을 정의하고 여정을 디자인하는 것부터 개인화된 콘텐츠를 만들고 결과를 분석하는 것에 이르기까지 전체 사용 사례를 한 곳에서 구현할 수 있습니다. 이 설명서에서는 Journey Optimizer를 효과적으로 사용하는 데 필요한 주요 역할과 그 책임 및 시작 방법을 설명합니다.

**중요 정보:** Adobe Journey Optimizer는 구체적인 책임에 따라 각기 역할을 정의합니다. 조직의 구조에 따라 한 개인이 여러 역할 또는 모든 역할을 수행할 수 있습니다.

## 역할 기반 빠른 시작 안내서

구현을 단순화하기 위해 Adobe Journey Optimizer는 전문성에 따라 작업을 특정 역할에 배치합니다. 각 역할은 원활한 고객 경험을 제공하는 데 필요한 필수 작업에 중점을 둡니다.

| 역할 | 주요 책임 | 주요 기술 | 일반적인 작업 |
|-------------------|----------------------------------|--------------------------------|-----------------------------------------------|
| **관리자** | 환경 설정 및 액세스 관리 | 시스템 구성, 사용자 관리, 보안 | 샌드박스 설정, 사용자 권한 관리, 채널 및 메시지 사전 설정 구성 |
| **데이터 엔지니어** | 고객 프로필 데이터 및 데이터 소스 | 데이터 모델링, XDM 스키마, 소스 커넥터 | 프로필 및 비즈니스 데이터를 스키마로 모델링하고, 소스 커넥터를 구성하고, 데이터 수집을 모니터링합니다. |
| **Developer** | 기술 구현 및 통합 | 모바일/웹 SDK, API, 이벤트 기반 아키텍처 | SDK 통합, 이벤트 구현, 사용자 지정 작업 엔드포인트 빌드 |
| **마케터** | 여정 디자인 및 개인화된 경험 | 여정 오케스트레이션, 콘텐츠 제작, 대상 타기팅 | 고객 여정 디자인, 메시지 만들기 및 개인화, 오퍼 및 의사 결정 구성 요소 관리, 대상자 정의 |

각 역할별로 Adobe Journey Optimizer 구현의 특정 단계를 담당해 체계적이고 효율적인 배포 프로세스를 보장합니다.

## 구현 순서와 역할 종속성

성공적인 Journey Optimizer 구현은 일반적으로 이 순서를 따르며, 이는 역할 간의 종속성을 반영합니다.

1. **관리자**: 환경 설정\
   관리자는 샌드박스를 구성하고, 액세스 제어를 설정하고, 채널 구성을 준비하여 기반을 다집니다. 다른 팀이 작업할 수 있도록 하려면 먼저 이 작업을 수행해야 합니다.
   * 개발, 스테이징 및 프로덕션 샌드박스 구성
   * 역할, 권한 및 OLAC(개체 수준 액세스 제어) 설정
   * 채널 구성(이메일, SMS, 푸시, 인앱, 웹, 콘텐츠 카드)
   * 하위 도메인 위임 및 IP 풀 설정
   * 금지 목록 및 동의 정책 구성

2. **데이터 엔지니어**: 데이터 기반 구축\
   데이터 엔지니어는 개인화를 지원하는 데이터 인프라를 구축하여 고객 데이터가 시스템으로 그리고 시스템을 통해 이동하는 방식을 정의합니다.
   * 고객 식별을 위한 ID 네임스페이스 만들기
   * XDM 스키마 디자인(프로필, 경험 이벤트, 관계형)
   * 데이터 세트를 설정하고 실시간 고객 프로필에 대해 활성화
   * 데이터 수집 구성(일괄 처리 및 스트리밍)
   * 복잡한 계산을 위한 계산된 속성 만들기
   * 여정에 대한 이벤트 및 데이터 소스 구성

3. **개발자**: 기술 통합을 구현합니다.\
   개발자는 SDK를 통합하고, 이벤트를 전송하고, API 엔드포인트를 구축하여 애플리케이션을 Journey Optimizer에 연결합니다. 이러한 구현을 통해 여정을 트리거하고 실행할 수 있습니다.
   * Mobile SDK(iOS/Android)와 푸시 알림 설정 통합
   * 웹 경험을 위한 Web SDK 구현
   * 애플리케이션에서 이벤트를 전송하여 여정 트리거
   * 외부 시스템 통합을 위한 사용자 정의 액션 엔드포인트 작성
   * Adobe Experience Platform Assurance를 사용한 테스트 구현

4. **마케터**: 고객 경험을 디자인하고 실행합니다.\
   마케터는 모든 기초적인 작업을 활용하여 고객 여정을 구축하고, 콘텐츠를 제작하며, 모든 채널에서 고객 경험을 최적화합니다.
   * 세분화, CSV 업로드 또는 대상자 컴포지션을 사용하여 대상자 작성
   * AI 어시스턴트 및 템플릿을 사용하여 개인화된 콘텐츠 디자인
   * 이벤트 및 대상자 트리거를 사용하여 다중 채널 여정 만들기
   * 실행 전 승인 워크플로를 통해 테스트
   * 보고서를 통해 얻은 인사이트를 바탕으로 성능 모니터링 및 최적화

**참고:** 이 순서가 일반적이지만 일부 활동은 동시에 발생할 수 있습니다. 예를 들어 데이터 엔지니어가 스키마를 구성하는 동안 개발자가 앱 통합 작업을 수행할 수 있습니다.

## 역할별 시작

각 역할은 역할의 초점에 맞게 조정된 구체적인 작업으로 시작됩니다. 이 초기 단계를 완료하면 전체 구현 프로세스에 대한 온보딩 및 정렬을 보다 원활하게 수행할 수 있습니다.

### 마케터 {#for-marketers}

마케터 또는 비즈니스 실무자로서, 모든 접점에서 개인적이고 상황에 맞는 경험을 제공하기 위해 고객 여정을 디자인합니다. 통합된 인터페이스를 사용하여 전체 사용 사례를 처음부터 끝까지 구현하게 됩니다.

**사용할 주요 기능:**

* **여정 오케스트레이션**: 채널 전반에서 발생하는 행동이나 이벤트에 따라 각 고객이 자신의 속도에 맞춰 진행할 수 있는 실시간 일대일 고객 참여를 구현합니다.
* **캠페인 오케스트레이션**: 시각적 캔버스를 사용하여 복잡하고 여러 단계를 거치는 배치 캠페인을 대규모로 디자인하고 자동화합니다. 계절 프로모션, 신제품 출시, 고객 관계 관리 등 브랜드 주도 캠페인에 적합합니다. 다중 엔터티 세분화를 활용하여 고객 데이터를 관련 엔터티(계정, 구매, 예약)와 연결하여 정확한 대상자를 생성합니다.
* **최신 메시지 디자이너**: 끌어다 놓기 인터페이스를 사용하여 이메일과 모바일 메시지를 디자인하고 개인화합니다. 기본 제공 템플릿을 수정하여 출시 기간 단축
* **의사 결정 관리**: 이메일 및 고객 접점에서 활용할 수 있는 중앙 집중식 라이브러리에서 오퍼, 자격 규칙과 기타 구성 요소를 생성하고 관리합니다.
* **자산 관리**: Adobe Experience Manager Assets Essentials가 Journey Optimizer에 완벽하게 통합되어 자산에 대한 액세스 및 게재가 간소화됩니다.
* **대상자 정의**: 관계형 쿼리를 사용하여 즉각적인 세분화가 가능한 온디맨드 대상자를 작성하고, 전송 전 정확한 대상자 수를 확인할 수 있습니다.
* **AI/ML 서비스**: 전송 시간 최적화 및 예측 참여 점수를 활용하여 고가치 고객을 타기팅하고 이탈 위험을 최소화합니다.

**시작 작업:** 사용 사례 템플릿과 마법사를 활용하여 새로운 고객 여정을 쉽게 만들고 배포합니다.

[마케터로 시작 →](path/marketer.md)

### 데이터 엔지니어 {#for-data-engineers}

데이터 아키텍트 또는 엔지니어로서, Journey Optimizer가 제공하는 경험을 지원하는 고객 프로필 데이터 및 기타 데이터 소스를 설정하고 유지 관리합니다.

**주요 책임 사항:**

* **고객 프로필 데이터**: 고객 프로필 데이터와 비즈니스 데이터를 스키마로 모델링하여 고객에 대한 완전하고 포괄적인 관점을 구축합니다.
* **관계형 데이터 모델링**: 오케스트레이션 캠페인의 경우, 고객 데이터를 계정, 구매, 구독, 예약과 같은 관련 엔터티와 연결하여 유연한 대상자 생성을 가능하게 하는 관계형 스키마를 디자인합니다.
* **소스 커넥터**: 웹, CRM, 오프라인 데이터 및 기타 소스에서 Adobe Experience Platform으로 데이터를 가져오도록 소스 커넥터를 구성합니다.
* **ID 확인**: ID 네임스페이스를 설정하여 프로필을 지속적으로 업데이트하고 고객을 세그먼트 및 여정에 실시간으로 추가하거나 제거합니다.
* **데이터 소스**: 고객 여정 전반에서 외부 신호를 실시간으로 수신하도록 데이터 소스를 구성합니다.
* **프로필 관리**: 실시간 고객 프로필에 대한 데이터 세트를 활성화하여 개인화된 경험을 제공합니다.
* **데이터 품질**: 데이터 수집을 모니터링하여 모든 데이터가 Journey Optimizer로 원활하게 전달되도록 합니다.

**시작 작업:** 첫 번째 고객 프로필 스키마를 모델링하고 데이터 수집을 시작하기 위한 소스 커넥터를 구성합니다.

[데이터 엔지니어로 시작하기 →](path/data-engineer.md)

### 관리자 대상 {#for-administrators}

관리자는 팀이 효율적이고 안전하게 작업할 수 있도록 Journey Optimizer 환경을 설정합니다.

**주요 책임:**

* **샌드박스**: 샌드박스를 만들고 관리하여 다른 사용자 그룹(개발, 테스트, 프로덕션)의 데이터와 여정을 분할합니다.
* **사용자 관리**: 다른 기능에 대한 액세스를 제어할 수 있도록 사용자 그룹 및 권한을 설정합니다.
* **채널 설정**: Journey Optimizer를 통해 배달된 메시지 및 에셋에서 일관된 브랜딩을 유지하도록 배달 채널 및 메시지 사전 설정을 구성합니다.
* **보안 및 거버넌스**: OLAC(개체 수준 액세스 제어) 적용, 동의 정책 구성 및 데이터 거버넌스 정책 구현
* **전달성**: 하위 도메인을 위임하고 IP 풀을 만들고 금지 목록 및 허용 목록을 관리합니다.
* **여정 구성**: 팀에 대한 여정 요소 및 구성을 설정합니다.

**다음으로 시작:** 샌드박스 및 사용자 권한을 구성한 다음 첫 번째 채널 구성과 메시지 사전 설정을 설정합니다.

[관리자로 시작하기 →](path/administrator.md)

### 개발자 대상 {#for-developers}

Journey Optimizer를 애플리케이션에 연결하는 기술 통합을 구현합니다.

**주요 책임:**

* Adobe Experience Platform Mobile SDK 통합(iOS/Android)
* 웹 경험 및 웹 푸시 알림용 웹 SDK 구현
* 푸시 알림 자격 증명 및 인증서 구성
* 애플리케이션에서 이벤트를 전송하여 여정 트리거
* Journey Optimizer가 사용자 지정 작업을 통해 호출하는 API 엔드포인트 빌드
* 웹, 모바일 및 기타 표면에 대한 코드 기반 경험 구현
* Adobe Experience Platform Assurance를 사용하여 구현 테스트 및 디버그
* 프로그래밍 방식 액세스를 위한 Journey Optimizer API 작업

**다음으로 시작:** Mobile 또는 Web SDK을 통합한 다음 첫 번째 이벤트를 구현하여 여정을 트리거합니다.

[개발자로 시작하기 →](path/developer.md)

## 역할 간 협업

Journey Optimizer 구현이 성공하려면 모든 역할에서 협업이 필요합니다. 각 역할은 원활한 고객 경험을 제공하기 위해 다른 사용자와 협력합니다.

>[!BEGINTABS]

>[!TAB 관리자]

**관리자** 액세스 및 구성을 관리하여 모든 팀을 활성화합니다. 작업 대상은 다음과 같습니다.

* **데이터 엔지니어**: 데이터 관리에 대한 권한을 부여하고, 샌드박스 액세스를 승인하고, 거버넌스 정책을 조정합니다.
* **개발자**: API 자격 증명을 제공하고, 테스트 환경을 설정하고, 채널 구성을 승인합니다.
* **마케터**: 여정/캠페인 권한을 할당하고, 채널을 구성하고, 테스트 환경을 지원합니다.

>[!TAB 데이터 엔지니어]

**데이터 엔지니어**&#x200B;는 모든 사용자에게 데이터 기반을 제공합니다. 작업 대상은 다음과 같습니다.

* **관리자**: 데이터 관리에 대한 권한을 요청하고 거버넌스 및 보존 정책을 조정합니다.
* **개발자**: XDM 스키마 및 이벤트 구조를 제공하고, 이벤트 페이로드 형식을 정의하고, 데이터 수집을 테스트합니다.
* **마케터**: 개인화에 대한 계산된 특성을 만들고, 대상을 빌드하고, 관계형 스키마를 구성합니다.

>[!TAB 개발자]

**개발자** 여정 기능을 제공하는 기술 통합을 구현합니다. 작업 대상은 다음과 같습니다.

* **데이터 엔지니어**: XDM 스키마 및 이벤트 구조를 가져오고, 데이터 수집 요구 사항에 맞게 조정하고, 이벤트 게재를 테스트합니다.
* **관리자**: API 사양, 권한 요청 및 자격 증명을 제공하고 테스트 전략을 조정합니다.
* **마케터**: 이벤트 트리거를 이해하고, 추적을 구현하고, 여정 테스트를 지원하며, 문제를 해결합니다.

>[!TAB 마케터]

**마케터** 고객 경험을 디자인하고 피드백을 제공합니다. 작업 대상은 다음과 같습니다.

* **데이터 엔지니어**: 계산된 특성을 요청하고, 대상 요구 사항을 조정하고, 데이터 품질에 대한 피드백을 제공합니다.
* **개발자**: 이벤트 트리거 조율, 구현 테스트, 추적 유효성 검증
* **관리자**: 채널 구성 요청, 기능 액세스 확인, 인에이블먼트 조율

>[!ENDTABS]

**모범 사례:** 다양한 분야의 사람들과 정기적으로 회의를 열어 팀 간의 우선 순위를 조정하고, 진행 상황을 공유하고, 방해 요소를 해결하세요.
