# 여정과 캠페인: 적합한 접근 방식 선택 {#journeys-vs-campaigns}

Adobe Journey Optimizer은 고객에게 도달하고 참여시킬 수 있는 세 가지 강력한 접근 방식을 제공합니다. 각 제품을 사용할 시기를 이해하는 것은 효과적인 마케팅 경험을 구축하는 데 중요합니다.

이 안내서는 특정 마케팅 요구 사항에 따라 **여정**, **캠페인**(액션 및 API 트리거) 및 **오케스트레이션된 캠페인** 중에서 선택할 수 있도록 도와줍니다.

## 빠른 비교 개요 {#quick-overview}

| 접근 방식 | 다음에 최적 | 실행 스타일 |
|----------|----------|-----------------|
| **여정** | 조건부 논리를 사용한 다단계 실시간 고객 경험 | 1:1 오케스트레이션 - 각 프로필의 속도에 맞게 조정 |
| **캠페인(액션 및 API)** | 단순, 예약 또는 트리거된 메시지 게재 | 일괄 처리 또는 API 트리거 - 모든 프로필 동시 실행 |
| **오케스트레이션된 캠페인** | 다중 엔티티 세그먼테이션이 있는 복잡한 일괄 처리 워크플로 | 일괄 처리 캔버스 - 모든 프로필이 함께 처리됨 |

## 세부 비교 {#detailed-comparison}

이 포괄적인 표를 사용하여 주요 차이점을 이해합니다.

| 기능 | 여정 | 캠페인(작업 및 API 트리거됨) | 오케스트레이션된 캠페인 |
|---------|----------|-----------------------------------|----------------------|
| **기본 목적** | 실시간 고객 컨텍스트를 사용한 여러 단계 1:1 오케스트레이션 | 대상에게 일회성 또는 반복 메시지 게재 | 복잡한 세분화 워크플로를 사용하는 여러 단계 일괄 캠페인 |
| **캔버스 형식** | 1:1 캔버스 - 각 프로필이 각자의 속도로 이동함 | 캔버스 없음 - 단일 작업 실행 | 일괄 처리 캔버스 - 모든 프로필이 함께 처리됨 |
| **실행 흐름** | 순차적 작업, 프로필이 여정 전체에서 상태를 유지 | 전체 대상에 대한 동시 실행 | 활동 및 전환을 포함하는 여러 단계 일괄 처리 워크플로 |
| **시작 메커니즘** | 이벤트, 대상, 자격 조건, 비즈니스 이벤트 | 수동 활성화, 예약됨 또는 API 트리거 | 배치 워크플로우의 예약된 실행 |
| **데이터 모델** | 실시간 프로필 + 이벤트 데이터 | 프로필 데이터 + API 페이로드 | 다중 엔티티 관계형 데이터(프로필, 제품, 스토어, 예약) |
| **세그먼테이션** | 사전 빌드된 대상 + 실시간 조건 | Experience Platform에서 사전 빌드된 대상 | 정확한 카운트로 캔버스 내에 구축된 온디맨드 대상자 |
| **프로필 처리** | 개별, 실시간(이벤트 발생 시) | 일괄 처리, 한 번에 | 일괄 처리, 다중 엔티티 지원 |
| **개인화** | 실시간 컨텍스트 데이터 + 프로필 속성 | 프로필 속성 + API 페이로드 데이터 | 정밀도 타깃팅을 위한 다중 엔티티 데이터 |
| **복잡성** | 분기, 대기 시간, 조건이 있는 여러 단계 | 단일 작업 또는 간단한 워크플로 | 세분화, 데이터 보강, 분할이 포함된 여러 단계 일괄 처리 워크플로 |
| **추천 항목** | 고객 라이프사이클 여정, 온보딩, 장바구니 포기 | 프로모션 캠페인, 뉴스레터, 공지, 트랜잭션 메시지 | 복잡한 시즌 캠페인, 여러 단계 프로모션, 제품 출시 |
| **시간** | 게시 후 지속적, 항상 활성 | 예약된 시작/종료 날짜 또는 API 트리거됨 | 일정에 따른 일괄 처리 실행 |
| **상태 관리** | 실시간 작업에 대한 고객 상태 유지 | 상태 비저장 실행 | 작업 테이블을 사용한 일괄 처리 |
| **다음의 경우에 사용** | 실시간 의사 결정 논리를 통해 여러 접점이 필요함 | 특정 시간에 대상자에게 보내는 간단한 메시지 | 복잡한 세그멘테이션, 다중 엔티티 데이터 또는 정확한 사전 전송 횟수 필요 |
| **고유 기능** | 실시간 반응, 대기 활동, 프로필 기반 게재 | 간단한 예약, API 트리거, 요금 제어 | 관계형 데이터 세트, 다중 엔티티 세그멘테이션, 정확한 카운트, 다중 레벨 전송 |

## 결정 안내서 {#decision-guide}

다음 의사 결정 트리에 따라 올바른 접근 방식을 선택하십시오.

### 1단계: 실행 요구 사항

**고객 행동에 대한 실시간 개별 응답**
→**여정 사용**
- 프로필이 원하는 속도로 이동해야 함
- 비헤이비어를 기반으로 한 조건부 논리
- 실시간 컨텍스트가 중요합니다.

대상자에게 **간단한 메시지를 전달하시겠습니까?**
→1&rbrace;작업 또는 API 트리거 캠페인 사용&#x200B;**&#x200B;**
- 모든 프로필이 동시에 메시지를 수신함
- API를 통해 예약되거나 트리거됨
- 복잡한 여러 단계 논리 필요 없음

**고급 세분화를 사용하는 복잡한 일괄 처리 워크플로우입니까?**
→ **오케스트레이션된 캠페인 사용**
- 다중 엔티티 데이터(제품, 스토어, 예약) 필요
- 정확한 사전 전송 횟수 필요
- 분할 및 데이터 보강 기능을 갖춘 여러 단계 일괄 처리

### 2단계: 선택 사항 유효성 검사

| 요구 사항 | 권장 접근 방식 | 이유 |
|-----------|---------------------|-----|
| 여러 단계 온보딩으로 새로운 고객 환영 | 여정 | 실시간 입력, 여러 접점, 조건부 경로 |
| 구독자에게 월별 뉴스레터 전송 | 액션 캠페인 | 대상자에 대한 간단한 예약 메시지 |
| 미리 알림 시퀀스가 있는 장바구니 포기 | 여정 | 실시간 트리거, 대기 시간, 조건부 후속 조치 |
| 모든 고객에 대한 프로모션 공지 | 액션 캠페인 | 일회성 메시지, 즉시 게재 |
| 비헤이비어에 따라 비활성 사용자 재참여 | 여정 | 대상 자격, 개인화된 경로에 의해 트리거됨 |
| 비즈니스 이벤트에 의해 트리거된 플래시 판매 | 여정(비즈니스 이벤트) | 여러 고객에게 영향을 주는 실시간 트리거 |
| 제품 카탈로그 통합으로 계절별 홍보 | 오케스트레이션된 캠페인 | 다중 엔티티 데이터, 복잡한 세분화, 정확한 카운트 |
| API 트리거된 트랜잭션 메시지 | API-트리거된 캠페인 | 외부 시스템 트리거, 즉각적인 전달 |
| 예약당 여러 수준 전송 | 오케스트레이션된 캠페인 | 다중 엔티티 관계, 예약당 하나의 메시지 |

## 주요 차이점 설명 {#key-distinctions}

### 여정: 1:1 실시간 오케스트레이션

**고유한 이유:**
- 각 프로필은 개별 상태 및 컨텍스트를 유지합니다
- 프로필이 원하는 속도로 시작 및 진행
- 행동 및 이벤트에 기반한 실시간 의사 결정
- 대기 활동은 개인화된 타이밍을 만듭니다.
- 조건부 분기는 프로필당 고유한 경로를 만듭니다.

**흐름 예:**

```
Customer A: Abandoned cart → Wait 2 hours → No purchase? → Send reminder → Purchased? → End
Customer B: Abandoned cart → Wait 2 hours → Already purchased → End immediately
```

각 고객은 자신의 작업을 기반으로 자체 여정 타임라인을 경험합니다.

[여정에 대해 자세히 알아보기](../building-journeys/journey.md)

### 캠페인: 간단한 일괄 처리 또는 트리거된 게재

**고유한 이유:**
- 모든 프로필이 동시에 동일하게 처리됨
- 상태 비저장 실행 - 유지 관리되는 컨텍스트 없음
- 간단한 예약 또는 API 트리거
- 브로드캐스트 통신에 이상적

**흐름 예:**

```
Monday 9 AM → Send newsletter to 100,000 subscribers → All receive simultaneously
```

모든 사람이 같은 시간에 같은 메시지를 받습니다.

**유형:**
- **작업 캠페인**: 예약된 게재(1회 또는 반복)
- **API 트리거 캠페인**: 외부 시스템의 API 호출을 통해 트리거됩니다.

[캠페인에 대해 자세히 알아보기](../campaigns/get-started-with-campaigns.md)

### 오케스트레이션된 캠페인: 일괄 처리 캔버스 워크플로

**고유한 이유:**
- 활동 및 전환이 포함된 일괄 처리 캔버스(여정 캔버스와 유사하지만 일괄 처리 중심)
- 다중 엔티티 관계형 데이터 지원(프로필 + 제품 + 스토어 + 예약)
- 캔버스 내 온디맨드 대상 구축
- 보내기 전 정확한 카운트(전송 전 가시성)
- 다중 레벨 전송(엔티티당 한 개의 메시지, 예: 예약당)
- 일괄 처리된 모든 프로필

**흐름 예:**

```
Query customers → Filter by purchase history → Split by region → 
Enrich with product data → Build segments → Send personalized offers → All in one batch execution
```

워크플로우의 복잡성과 일괄 캠페인 실행을 결합합니다.

[오케스트레이션된 캠페인에 대해 자세히 알아보기](../orchestrated/gs-orchestrated-campaigns.md)

## 사용 사례 예 {#use-cases}

### 여정 사용 사례

&#x200B;* **장바구니 포기 복구**: 장바구니 추가 이벤트에 의해 트리거되고, 체크아웃할 때까지 기다리며, 구매가 없을 경우 미리 알림을 보냅니다.
&#x200B;* **고객 온보딩**: 프로필 데이터를 기반으로 개인화된 콘텐츠가 포함된 여러 단계 시작 시리즈
&#x200B;* **충성도 계층 업그레이드**: 고객이 새 계층에 도달하면 트리거되어 축하와 혜택을 보냅니다.
&#x200B;* **생일 캠페인**: 생일, 맞춤 오퍼에 따른 항목
&#x200B;* **다시 참여**: 대상 자격(비활성), 점진적 전달을 통해 트리거됩니다.

### 캠페인 사용 사례(작업 및 API 트리거됨)

**작업 캠페인:**
&#x200B;* **월별 뉴스레터**: 구독자 세그먼트에 예약된 일괄 게재
&#x200B;* **프로모션 공지**: 타겟 대상자에 대한 시간에 민감한 오퍼
&#x200B;* **제품 출시**: 모든 고객에게 조정된 발표
&#x200B;* **계절별 인사**: 특정 날짜에 대한 휴일 메시지

**API 트리거 캠페인:**
&#x200B;* **주문 확인**: 구매 후 전자 상거래 시스템에서 트리거됩니다.
&#x200B;* **배송 알림**: 물류 시스템에 의해 트리거됨
&#x200B;* **계정 경고**: 사기 행위 감지 시스템에 의해 트리거됨
&#x200B;* **암호 재설정**: 응용 프로그램의 사용자 동작에 의해 트리거됩니다.

### 조정된 캠페인 사용 사례

&#x200B;* **카탈로그 통합을 통한 시즌 프로모션**: 제품 카탈로그를 쿼리하고, 적격 고객을 식별하고, 환경 설정에 따라 세그먼트화하고, 개인화된 제품 추천을 보냅니다.
&#x200B;* **스토어 특정 캠페인**: 스토어 인벤토리 데이터를 사용하여 특정 스토어 위치 근처의 고객을 타겟팅합니다.
&#x200B;* **다중 예약 통신**: 예약당 하나의 메시지 보내기(호텔 예약, 항공편 예약)
&#x200B;* **복잡한 세그먼트 오케스트레이션**: 여러 데이터 소스의 데이터 보강 기능을 사용하여 단계별로 대상을 빌드합니다.
&#x200B;* **미리 보내기 유효성 검사**: 주요 캠페인을 시작하기 전에 정확한 수신자 수를 가져옵니다.

## 일반적인 질문 {#common-questions}

**Q: 마케팅 전략에 여정과 캠페인을 결합할 수 있습니까?**

A: 당연하지! 대부분의 조직은 서로 다른 시나리오에 대해 세 가지 접근 방식을 모두 사용합니다.
- 행동 및 실시간 참여에 대한 여정
- 예약된 브로드캐스트 통신에 대한 작업 캠페인
- 트랜잭션 메시지를 위한 API 트리거 캠페인
- 복잡한 데이터 집약적 일괄 처리 캠페인을 위한 통합 캠페인

**Q: 캠페인을 여정으로 전환하거나 그 반대로 전환할 수 있습니까?**

A: 아니요. 적절한 형식으로 경험을 다시 빌드해야 합니다. 하지만 콘텐츠, 대상 및 논리 개념을 재사용할 수 있습니다.

**Q: 어떤 접근 방식을 더 쉽게 작성할 수 있습니까?**

답변: 작업 캠페인은 일반적으로 가장 간단한(대상에 대한 단일 메시지) 캠페인, API 트리거 캠페인, 여정(다중 단계 로직으로 더 복잡함) 및 오케스트레이션된 캠페인(캔버스 워크플로우 및 다중 엔티티 기능으로 인해 가장 복잡함)입니다.

**Q: 대규모 대상자에게 더 적합한 확장 방법은 무엇입니까?**

A: 세 가지 모두 확장이 가능하지만,
- **대상자 여정 읽기** 및 **작업 캠페인**&#x200B;은(는) 일괄 처리 대상자에 맞게 최적화되었습니다.
- 대규모 데이터 세트를 포함하는 복잡한 세분화에서 **오케스트레이션된 캠페인** excel
- **단일 여정**&#x200B;은(는) 개별적으로 프로필을 처리하므로 스케일은 이벤트 볼륨에 따라 다릅니다

**Q: 여정 및 캠페인에서 동일한 대상을 사용할 수 있습니까?**

A: 예. Adobe Experience Platform에서 만든 대상자는 세 가지 접근 방식 모두에서 사용할 수 있습니다.
