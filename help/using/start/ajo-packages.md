# Adobe Journey Optimizer 패키지 및 기능 {#ajo-packages}

>[!BEGINSHADEBOX]

**이 페이지에서:** 기본 오퍼, 채널 추가 기능 및 Decisioning 추가 기능 전체에 걸쳐 모듈식 Adobe Journey Optimizer 패키징이 작동하는 방식을 알아봅니다. 이를 통해 참여 사용 사례와 예산에 맞는 조합을 선택할 수 있습니다.

>[!ENDSHADEBOX]

[!DNL Adobe Journey Optimizer]은(는) 모듈식 패키징 모델을 사용합니다. 기본 사용 사례와 일치하는 기본 오퍼로 시작한 다음 필요한 채널 및 고급 기능을 추가합니다.

>[!TIP]
>
>**현재 사용 중인 모델을 알 수 없습니까?** 모듈식 패키징 모델로 [!DNL Adobe Journey Optimizer]을(를) 구입한 경우 이 페이지에 설명된 기본 오퍼 및 추가 기능이 적용됩니다. 계약에 **Select**, **Prime** 또는 **Ultimate**&#x200B;이(가) 참조된 경우 기존 패키징 모델에 있습니다. 현재 사용 권한을 확인하려면 Adobe 담당자에게 문의하세요.

>[!NOTE]
>
>패키지 가용성 및 포함된 기능은 계약, 선택한 추가 기능 및 지역별 가용성에 따라 달라질 수 있습니다. 조직 고유의 자세한 내용은 Adobe 담당자에게 문의하십시오.

## 빠른 답변 {#quick-answers}

대부분의 사람들은 더 읽기 전에 두 가지 질문을 합니다. [전체 FAQ](#faq)는 페이지 하단에 있습니다.

+++**모든 기본 오퍼에 모든 채널이 포함되어 있습니까?**

아니요. [!DNL Adobe Journey Optimizer]은(는) 모듈형 모델을 사용합니다. 기본 오퍼는 오케스트레이션 기능(캠페인, 여정 또는 둘 다)을 결정하고 채널 추가 기능은 참여할 수 있는 메시징 표면을 결정합니다. 사용 사례와 예산에 맞는 조합을 선택합니다.

+++

+++**캠페인과 여정 간의 차이점은 무엇입니까?**

**캠페인**&#x200B;은(는) 대상을 기반으로 하며 마케터가 계획되어 있습니다. 대상을 정의하고 메시지를 만들고 일괄 전송으로 예약하거나 트리거합니다. 이는 홍보 활동, 뉴스레터 및 다단계 대상 워크플로우에 가장 적합합니다.

**여정**&#x200B;은(는) 실시간 및 이벤트 기반이며, 발생할 때마다 개별 고객 행동에 반응하여 터치포인트 간에 1:1 경험을 조정합니다. 이 시퀀스는 온보딩 플로우, 구매 후 시퀀스 및 실시간 트리거 메시지에 가장 적합합니다.

**캠페인 및 여정**&#x200B;은(는) 단일 라이선스에서 두 기능을 모두 제공합니다.

+++

## 1단계 — 기본 오퍼 식별 {#base-offers}

>[!NOTE]
>
>**Journey Optimizer을 평가 중?** 이 단계에서는 참여 전략에 맞는 기본 오퍼를 선택합니다. **이미 고객입니까?** 보유하고 있는 기본 오퍼와 수행하는 작업을 확인하는 방법입니다.

세 가지 기본 오퍼를 사용할 수 있습니다. 각각의 방식은 고객을 끌어모으는 다른 방식과 일치합니다.

| 기본 오퍼 | 다음에 최적 | 핵심 동작 |
|-----------|---------|--------------|
| **Journey Optimizer - 캠페인** | 일괄 처리, 마케터 계획 지원 | 대상자 기반, 예약된 오케스트레이션. 일괄 처리 참여 및 마케터 계획 전달을 위한 단일 또는 다중 단계 캠페인 워크플로우입니다. |
| **Journey Optimizer - 여정** | 실시간 고객 참여 | 이벤트 기반, 1:1 오케스트레이션. 실시간 및 예약된 여정 전달을 모두 지원합니다. |
| **Journey Optimizer - 캠페인 및 여정** | 두 가지 모두를 필요로 하는 고객 | 대상자 기반 캠페인 오케스트레이션과 실시간 여정 오케스트레이션을 결합합니다. |

>[!IMPORTANT]
>
>**저장소 사용 권한이 패키지별로 다릅니다.** 총 데이터 볼륨 권한은 기본 오퍼에 따라 다릅니다. **캠페인** 고객은 주소 지정 프로필당 **15KB**&#x200B;을(를) 받고, **여정** 및 **캠페인 및 여정** 고객은 주소 지정 프로필당 **75KB**&#x200B;을(를) 받습니다. 패키지를 선택하거나 확인할 때에서 이를 팩터링합니다.

### 캠페인과 여정 — 차이점은 무엇입니까? {#campaigns-vs-journeys}

| | Journey Optimizer - 캠페인 | JOURNEY OPTIMIZER - 여정 | Journey Optimizer - 캠페인 및 여정 |
|--|:-----------------------------:|:----------------------------:|:----------------------------------------:|
| 대상 기반 일괄 처리 오케스트레이션 | ✓ | 제한적¹ | ✓ |
| 실시간 이벤트 기반 오케스트레이션 | — | ✓ | ✓ |
| 트랜잭션 메시지(이메일, 푸시, SMS) | ✓ | ✓ | ✓ |
| 채널 추가 기능 사용 가능 | ✓ | ✓ | ✓ |
| 의사 결정 추가 기능 사용 가능 | ✓ | ✓ | ✓ |

¹ **Journey Optimizer - 여정**&#x200B;에서 대상 기반 오케스트레이션은 독립 실행형 일괄 처리 캠페인이 아닌 여정 사용 사례 내에서만 지원됩니다.

## 2단계 — 필요한 채널 추가 {#channel-addons}

채널은 기본 오퍼에 번들로 제공되지 않습니다. 참여 전략에 맞는 채널 추가 기능 또는 추가 기능 번들을 선택합니다.

>[!BEGINTABS]

>[!TAB ✉️ 아웃바운드 게재]

아웃바운드 메시징 채널을 통해 대상에게 도달합니다.

**포함:** 전자 메일, 푸시 알림, DM

**일반적인 사용 사례:** 프로모션 이메일, 트랜잭션 푸시 알림, 실제 메일 캠페인

**지원되는 표면:** 받은 편지함, 모바일 장치 잠금 화면/알림 트레이, 우편 주소

새 인스턴스에 대한 IP 준비 지원을 위한 전달성 기본 사항이 포함됩니다. 지속적인 모니터링과 컨설턴트 주도 지원을 위해 세 가지 **전달성 패키지**(Essentials, Enhanced 및 Plus)를 별도의 추가 기능으로 사용할 수 있습니다. [전달성에 대해 알아보기](../reports/deliverability.md)

>[!TAB 📱 모바일]

인세션 및 지속적인 모바일 경험을 통해 앱 사용자를 참여시킵니다.

**포함:** 인앱 메시지, 푸시 알림, 콘텐츠 카드, 모바일 표면용 코드 기반 채널

**일반적인 사용 사례:** 온보딩 흐름, 기능 공지, 충성도 누지, 실시간 인앱 오퍼

**지원되는 표면:** 모바일 앱 화면, 알림 트레이, 영구 컨텐츠 슬롯, SDK을 통한 사용자 지정 앱 표면

>[!TAB 🌐 웹]

코드를 배포하지 않고 웹 경험을 개인화할 수 있습니다.

**웹 채널(시각적 및 비시각적 편집기), 웹 표면에 대한 코드 기반 채널 포함:**

**일반적인 사용 사례:** 홈페이지 배너, 랜딩 페이지 개인화, A/B 테스트, API를 통한 헤드리스 웹 개인화

**지원되는 표면:** 브라우저 페이지, 단일 페이지 앱(SPA), Headless 웹 끝점

>[!TAB 📦 모든 채널]

**모든 채널** 추가 기능 번들은 아웃바운드 게재 + 모바일 + 웹을 한 번에 구매합니다.

**포함 항목:** 세 가지 개별 추가 기능(이메일, 푸시 알림, DM, 인앱 메시지, 콘텐츠 카드, 웹 채널 및 코드 기반 채널)의 모든 채널

**일반적인 사용 사례:** 아웃바운드, 모바일 앱 및 웹에 걸친 조정된 옴니채널 프로그램(예: 인앱 메시지 및 개인화된 웹 배너를 포함하는 전자 메일 후속 캠페인)

**지원되는 표면:** 아웃바운드 게재, 모바일 및 웹 추가 기능의 모든 표면

>[!NOTE]
>
>**모든 채널** 번들은 일반적으로 Outbound Delivery, Mobile 및 Web을 개별적으로 라이선스하는 것보다 비용 효율적입니다. 세 가지 모두 사용하려면 Adobe 담당자에게 번들 가격과 개별 추가 기능을 비교하도록 요청하십시오.

>[!ENDTABS]

### 추가 채널 {#additional-channels}

일부 채널이 단일 채널 추가 기능에 연결되지 않았습니다. 사용 가능 여부는 라이센스 구성, 계약 및 타사 비즈니스 계정의 파트너 채널에 따라 다릅니다.

**📲SMS/MMS** - 휴대폰 번호로 문자 및 멀티미디어 메시지를 보냅니다. 트랜잭션 SMS(실시간, 이벤트 트리거)는 모든 기본 오퍼에서 지원됩니다. SMS/MMS의 광범위한 마케팅 사용은 라이선스가 부여된 구성에 따라 달라집니다. [모바일 메시지를 보내는 방법 알아보기](../mobile/get-started-mobile.md)

**💬WhatsApp** — WhatsApp 비즈니스를 통해 메시지를 보냅니다. WhatsApp 비즈니스 계정이 필요하며 사용 가능 여부는 계약 및 라이선스가 부여된 구성에 따라 다릅니다. [WhatsApp 사용 방법 알아보기](../whatsapp/get-started-whatsapp.md)

>[!NOTE]
>
>Adobe 담당자에게 조직의 SMS/MMS 및 WhatsApp 가용성을 확인합니다. 이러한 채널은 표준 아웃바운드 게재, 모바일 또는 웹 추가 기능의 일부가 아닙니다.

## 3단계 — 고급 기능 추가 {#advanced-addons}

### 결정 {#decisioning-addon}

**Decisioning** 추가 기능을 사용하면 모든 채널에서 실시간으로 각 프로필에 대한 최상의 오퍼, 작업 또는 경험을 정의, 관리 및 제공할 수 있습니다.

**잠금 해제하는 항목:**
- 자격 규칙, 순위 논리 및 제약 조건을 사용한 실시간 오퍼 선택
- 자동으로 오퍼 성능을 최적화하는 AI 기반 등급 모델
- 여정, 캠페인 및 코드 기반 환경에서 사용할 수 있는 의사 결정 정책

**사용권 계약에 따라 세 가지 기본 오퍼를 모두 사용할 수 있습니다.** [의사 결정 사용 방법 알아보기](../experience-decisioning/gs-experience-decisioning.md) | [AI 모델에 대해 알아보기](../offers/ranking/ai-models.md)

## 개요 비교 {#comparison-matrix}

기능이 두 개의 테이블로 분할되어 각 테이블을 쉽게 검색할 수 있습니다. 오케스트레이션 기능은 **기본 오퍼**&#x200B;에 의해 결정되지만 채널 및 고급 기능은 **추가 기능**&#x200B;에 의해 잠금 해제됩니다.

### 오케스트레이션 기능 {#orchestration-capabilities}

기본 오퍼에 의해 결정됩니다.

| 기능 | 캠페인 | 여정 | 캠페인 및 여정 |
|-----------|:---------:|:--------:|:--------------------:|
| 트랜잭션 메시지(이메일, 푸시, SMS) | ✓ | ✓ | ✓ |
| 캠페인 일괄 처리 | ✓ | — | ✓ |
| 오케스트레이션된 캠페인 _(이메일, SMS, 푸시, DM만 해당)_ | ✓ | — | ✓ |
| 자동화된 여정 | — | ✓ | ✓ |
| 실시간 이벤트 트리거 | — | ✓ | ✓ |

### 채널 및 고급 기능 {#channel-capabilities}

대부분의 채널은 세 가지 기본 오퍼 모두에서 사용할 수 있으며 나열된 추가 기능이 필요합니다. SMS/MMS 및 WhatsApp과 같은 몇 가지 기능은 사용 허가된 구성에 따라 다릅니다.

| 기능 | 가용성 | 추가 기능 필요 |
|-----------|-------------|----------------|
| 이메일 | 모든 기본 오퍼 | 아웃바운드 게재 |
| 푸시 알림 | 모든 기본 오퍼 | 아웃바운드 게재 |
| DM | 모든 기본 오퍼 | 아웃바운드 게재 |
| SMS / MMS | 라이센스 구성 기반 | 라이센스 구성 기반 |
| 인앱 메시징 | 모든 기본 오퍼 | 모바일 |
| 콘텐츠 카드 | 모든 기본 오퍼 | 모바일 |
| 웹 채널 | 모든 기본 오퍼 | 웹 |
| 코드 기반 경험 | 모든 기본 오퍼 | 모바일 또는 웹 |
| WhatsApp | 라이센스 구성 기반 | WhatsApp |
| 결정 | 라이센스에 따라 다름 | 결정 |
| AI 기반 순위 | 라이센스에 따라 다름 | 결정 |

>[!NOTE]
>
>**트랜잭션 메시지와 채널 추가 기능 비교** 기본 트랜잭션 메시지(이벤트 트리거 이메일, 푸시 및 SMS)는 모든 기본 오퍼에 포함됩니다. **아웃바운드 게재**, **모바일** 및 **웹** 추가 기능으로 위에 나열된 추가 표면과 함께 해당 채널의 전체 마케팅 및 캠페인 사용을 잠금 해제합니다.

## 자주 묻는 질문 {#faq}

+++**오케스트레이션된 캠페인에서 지원되는 채널은 무엇입니까?**

오케스트레이션된 캠페인(Campaign Orchestration 기능을 사용하는 다단계 대상 워크플로)은 **이메일, SMS, 푸시 알림 및 DM**&#x200B;만 지원합니다. 웹, 인앱, 코드 기반 및 콘텐츠 카드 채널은 오케스트레이션된 캠페인 워크플로우에서 지원되지 않습니다.

+++

+++**의사 결정이 모든 구성에 포함되어 있습니까?**

아니요. Decisioning은 고유한 고급 기능 추가 기능이며 기본적으로 기본 오퍼에 포함되지 않습니다. Decisioning을 구성에 추가하려면 Adobe 담당자에게 문의하십시오.

+++

+++**Select, Prime 또는 Ultimate에 대해 들었습니다. 아직 현재 패키징 모델입니까?**

[!DNL Adobe Journey Optimizer]은(는) 기본 오퍼(캠페인, 여정, 캠페인 및 여정) 및 추가 기능을 기반으로 구축된 모듈식 패키징 모델을 통해 제공됩니다. Select, Prime 또는 Ultimate 용어를 사용하는 기존 고객의 경우 아래 [레거시 패키징](#legacy-packaging)에서 기능 참조를 참조하거나 Adobe 담당자에게 문의하여 현재 사용 권한을 확인하십시오.

+++

## 레거시 패키징 - Select, Prime 및 Ultimate {#legacy-packaging}

위의 섹션에서는 현재 모듈식 패키징 모델을 설명합니다. 계약에서 여전히 **Select**, **Prime** 또는 **Ultimate** 용어를 사용하는 경우 아래 참조를 확장하여 각 레거시 패키지에 포함된 내용을 확인하세요. 모듈식 모델로 이동하려면 Adobe 담당자에게 문의하십시오.

+++**기존 패키징 참조 표시**

**패키지 요약**

**선택** - 일괄 처리 및 트랜잭션 메시지를 시작하는 조직에 가장 적합합니다.

- 예약된 일괄 캠페인 및 트랜잭션 메시지
- 핵심 캠페인 및 여정 실행
- 이메일, SMS, 푸시 및 사용자 정의 작업 채널 기초
- 표준 오케스트레이션 보호

**Prime** — Select의 모든 기능과 실시간 오케스트레이션 및 인바운드 채널을 포함합니다.

- 이벤트가 트리거되는 실시간 여정 오케스트레이션
- 인바운드 채널: 웹, 인앱 메시지, 코드 기반 경험, 콘텐츠 카드 및 DM
- 고급 대상자 세분화 및 타겟팅

**Ultimate** — Prime의 모든 기능과 더불어 의사 결정 및 고급 최적화가 포함됩니다.

- 실시간 오퍼 의사 결정 및 개인화
- AI 기반 순위 및 최적화 모델
- 고급 보고 및 실험 기능
