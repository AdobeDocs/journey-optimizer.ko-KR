# Adobe Journey Optimizer 패키지 및 기능 {#ajo-packages-v2}

[!DNL Adobe Journey Optimizer]은(는) 모듈식 패키징 모델을 사용합니다. 기본 사용 사례와 일치하는 기본 오퍼로 시작한 다음 필요한 채널 및 고급 기능을 추가합니다.

>[!NOTE]
>
>패키지 가용성 및 포함된 기능은 계약, 선택한 추가 기능 및 지역별 가용성에 따라 달라질 수 있습니다. 조직 고유의 자세한 내용은 Adobe 담당자에게 문의하십시오.

## 1단계 — 기본 오퍼 선택 {#base-offers}

세 가지 기본 오퍼를 사용할 수 있습니다. 고객을 주로 참여시키는 방식에 일치하는 항목을 선택합니다.

| 기본 오퍼 | 다음에 최적 | 핵심 동작 |
|-----------|---------|--------------|
| **Journey Optimizer - 캠페인** | 일괄 처리, 마케터 계획 지원 | 대상자 기반, 예약된 오케스트레이션. 관계형 데이터 저장소를 사용하는 단일 또는 다중 단계 캠페인 워크플로우입니다. |
| **Journey Optimizer - 여정** | 실시간 고객 참여 | 이벤트 기반, 1:1 오케스트레이션. 스트리밍 및 일괄 여정 처리를 모두 지원합니다. |
| **Journey Optimizer - 캠페인 및 여정** | 두 가지 모두를 필요로 하는 고객 | 대상자 기반 캠페인 오케스트레이션과 실시간 여정 오케스트레이션을 결합합니다. |

### 캠페인과 여정 — 차이점은 무엇입니까? {#campaigns-vs-journeys}

| | Journey Optimizer - 캠페인 | JOURNEY OPTIMIZER - 여정 | Journey Optimizer - 캠페인 및 여정 |
|--|:-----------------------------:|:----------------------------:|:----------------------------------------:|
| 대상 기반 일괄 처리 오케스트레이션 | ✓ | 여정 사용 사례로 제한 | ✓ |
| 실시간 이벤트 기반 오케스트레이션 | — | ✓ | ✓ |
| 트랜잭션 메시지(이메일, 푸시, SMS) | ✓ | ✓ | ✓ |
| 채널 추가 기능 사용 가능 | ✓ | ✓ | ✓ |
| 의사 결정 추가 기능 사용 가능 | ✓ | ✓ | ✓ |

>[!NOTE]
>
>총 데이터 볼륨 권리 유형은 다릅니다. **캠페인** 고객은 주소 지정 프로필당 15KB를 받고, **여정** 및 **캠페인 및 여정** 고객은 주소 지정 프로필당 75KB를 받습니다.

## 2단계 — 필요한 채널 추가 {#channel-addons}

채널은 기본 오퍼에 번들로 제공되지 않습니다. 참여 전략에 맞는 채널 추가 기능 또는 추가 기능 번들을 선택합니다.

>[!BEGINTABS]

>[!TAB 아웃바운드 게재]

아웃바운드 메시징 채널을 통해 대상에게 도달합니다.

**포함:** 전자 메일, 푸시 알림, DM

**일반적인 사용 사례:** 프로모션 이메일, 트랜잭션 푸시 알림, 실제 메일 캠페인

**지원되는 표면:** 받은 편지함, 모바일 장치 잠금 화면/알림 트레이, 우편 주소

새 인스턴스에 대한 IP 준비 지원을 위한 전달성 기본 사항이 포함됩니다. [전달성에 대해 알아보기](../reports/deliverability.md)

>[!TAB 모바일]

인세션 및 지속적인 모바일 경험을 통해 앱 사용자를 참여시킵니다.

**포함:** 인앱 메시지, 푸시 알림, 콘텐츠 카드, 모바일 표면용 코드 기반 채널

**일반적인 사용 사례:** 온보딩 흐름, 기능 공지, 충성도 누지, 실시간 인앱 오퍼

**지원되는 표면:** 모바일 앱 화면, 알림 트레이, 영구 컨텐츠 슬롯, SDK을 통한 사용자 지정 앱 표면

>[!TAB 웹]

코드를 배포하지 않고 웹 경험을 개인화할 수 있습니다.

**웹 채널(시각적 및 비시각적 편집기), 웹 표면에 대한 코드 기반 채널 포함:**

**일반적인 사용 사례:** 홈페이지 배너, 랜딩 페이지 개인화, A/B 테스트, API를 통한 헤드리스 웹 개인화

**지원되는 표면:** 브라우저 페이지, 단일 페이지 앱(SPA), Headless 웹 끝점

>[!TAB 모든 채널]

**모든 채널** 추가 기능 번들은 아웃바운드 게재 + 모바일 + 웹을 한 번에 구매합니다.

아웃바운드, 모바일 앱 및 웹 표면에 걸쳐 완전한 옴니채널 커버리지를 필요로 하는 조직에 가장 적합합니다.

>[!ENDTABS]

**WhatsApp**&#x200B;은(는) WhatsApp Business를 통해 메시지를 전송해야 하는 고객을 위해 별도의 추가 기능으로 사용할 수 있습니다. [WhatsApp 사용 방법 알아보기](../whatsapp/get-started-whatsapp.md)

## 3단계 — 고급 기능 추가 {#advanced-addons}

### 결정 {#decisioning-addon}

**Decisioning** 추가 기능을 사용하면 모든 채널에서 실시간으로 각 프로필에 대한 최상의 오퍼, 작업 또는 경험을 정의, 관리 및 제공할 수 있습니다.

**잠금 해제하는 항목:**
- 자격 규칙, 순위 논리 및 제약 조건을 사용한 실시간 오퍼 선택
- 자동으로 오퍼 성능을 최적화하는 AI 기반 등급 모델
- 여정, 캠페인 및 코드 기반 환경에서 사용할 수 있는 의사 결정 정책

**사용권 계약에 따라 세 가지 기본 오퍼를 모두 사용할 수 있습니다.** [의사 결정 사용 방법 알아보기](../experience-decisioning/gs-experience-decisioning.md) | [AI 모델에 대해 알아보기](../offers/ranking/ai-models.md)

## 자주 묻는 질문 {#faq}

+++**모든 기본 오퍼에 모든 채널이 포함되어 있습니까?**

아니요. [!DNL Adobe Journey Optimizer]은(는) 모듈형 모델을 사용합니다. 기본 오퍼는 오케스트레이션 기능(캠페인, 여정 또는 둘 다)을 결정하고 채널 추가 기능은 참여할 수 있는 메시징 표면을 결정합니다. 사용 사례와 예산에 맞는 조합을 선택합니다.

+++

+++**캠페인과 여정 간의 차이점은 무엇입니까?**

**캠페인**&#x200B;은(는) 대상을 기반으로 하며 마케터가 계획되어 있습니다. 대상을 정의하고 메시지를 만들고 일괄 전송으로 예약하거나 트리거합니다. 이는 홍보 활동, 뉴스레터 및 다단계 대상 워크플로우에 가장 적합합니다.

**여정**&#x200B;은(는) 실시간 및 이벤트 기반이며, 발생할 때마다 개별 고객 행동에 반응하여 터치포인트 간에 1:1 경험을 조정합니다. 이 시퀀스는 온보딩 플로우, 구매 후 시퀀스 및 실시간 트리거 메시지에 가장 적합합니다.

**캠페인 및 여정**&#x200B;은(는) 단일 라이선스에서 두 기능을 모두 제공합니다.

+++

+++**오케스트레이션된 캠페인에서 지원되는 채널은 무엇입니까?**

오케스트레이션된 캠페인(Campaign Orchestration 기능을 사용하는 다단계 대상 워크플로)은 **이메일, SMS, 푸시 알림 및 DM**&#x200B;만 지원합니다. 웹, 인앱, 코드 기반 및 콘텐츠 카드 채널은 오케스트레이션된 캠페인 워크플로우에서 지원되지 않습니다.

+++

+++**의사 결정이 모든 구성에 포함되어 있습니까?**

아니요. Decisioning은 고유한 고급 기능 추가 기능이며 기본적으로 기본 오퍼에 포함되지 않습니다. Decisioning을 구성에 추가하려면 Adobe 담당자에게 문의하십시오.

+++

+++**Select, Prime 또는 Ultimate에 대해 들었습니다. 아직 현재 패키징 모델입니까?**

[!DNL Adobe Journey Optimizer]은(는) 기본 오퍼(캠페인, 여정, 캠페인 및 여정) 및 추가 기능을 기반으로 구축된 모듈식 패키징 모델을 통해 제공됩니다. Select, Prime 또는 Ultimate 용어를 사용하는 기존 고객이고 현재 사용 권한에 대한 질문이 있는 경우 Adobe 담당자에게 문의하십시오.

+++
