# 여정 속성 속성 {#journey-properties}

[단순 식 편집기](../conditions.md#about_condition)와 [고급 식 편집기](../expression/expressionadvanced.md)의 **이벤트** 및 **데이터 원본** 범주 아래에서 **여정 속성** 범주에 액세스할 수 있습니다. 이 카테고리에는 지정된 프로필의 여정과 관련된 기술 필드가 포함되어 있습니다. 이는 여정 ID 또는 발생한 특정 오류와 같은 라이브 여정에서 시스템이 검색한 정보입니다.

![](../assets/journey-properties.png)

여기에는 다음과 같은 정보가 포함됩니다.

* 여정 버전: 여정 uid, 여정 버전 uid, 인스턴스 uid 등
* 오류: 데이터 가져오기, 작업 실행 등
* 현재 단계, 마지막 현재 단계 등
* 삭제된 프로필

  필드 목록을 [이 섹션에서 사용 가능](#journey-properties-fields).

이러한 필드를 사용하여 표현식을 작성할 수 있습니다. 여정 실행 중에 여정에서 직접 값을 검색합니다.

다음은 사용 사례의 몇 가지 예입니다.

* **삭제된 프로필 기록**: 로깅 목적으로 최대 가용량 규칙에 의해 메시지에서 제외된 모든 프로필을 서드파티 시스템으로 보낼 수 있습니다. 이 경우 시간 초과 및 오류 발생 시 경로를 설정하고 특정 오류 유형(예: &quot;최대 가용량 규칙으로 사용자 삭제&quot;)을 필터링할 조건을 추가합니다. 그런 다음 사용자 지정 작업을 통해 삭제된 프로필을 타사 시스템으로 푸시할 수 있습니다.

* **오류 발생 시 알림 보내기**: 메시지에 오류가 발생할 때마다 서드파티 시스템으로 알림을 보낼 수 있습니다. 이를 위해 오류가 발생한 경우 경로를 설정하고, 조건 및 사용자 지정 작업을 추가합니다. 예를 들어 발생한 오류에 대한 설명과 함께 Slack 채널에서 알림을 보낼 수 있습니다.

* **보고에서 오류 세분화** : 오류 메시지에 대해 경로가 하나뿐인 대신 오류 유형별로 조건을 정의할 수 있습니다. 이렇게 하면 보고를 세분화하고 모든 오류 유형 데이터를 볼 수 있습니다.

이해를 돕기 위해 이 정보를 이 페이지의 설명서와 통합해야 합니다. 두 소스 모두 독립적으로 사용하기 위한 것은 아닙니다. 이 페이지에서는 기능에 대해 설명하지만, 용어, 의도, 적용 가능성 및 제약 조건을 명확히 하는 데 도움이 되는 추가 컨텍스트를 제공합니다.

* **TL;DR:** 이 페이지에서는 로깅, 경고 및 오류별 보고를 위해 표현식을 작성하는 데 사용할 수 있는 라이브 여정 인스턴스(ID, 오류, 현재/이전 노드, 경과 시간)에 대한 기술 필드 집합인 표현식 편집기의 여정 속성 범주에 대해 설명합니다.

**의도:**

* 단순 또는 고급 표현식 편집기의 여정 속성 필드에 액세스하여 라이브 여정 메타데이터를 참조합니다.
* 삭제된 프로필을 오류 유형별로 필터링하여 서드파티 로깅 시스템으로 라우팅하는 조건을 작성합니다
* 사용자 지정 작업에서 마지막 오류 코드 및 노드 이름을 참조하여 외부 채널(예: Slack)에 오류 알림을 보냅니다.
* `lastNodeTypeInError` 및 `lastErrorCode`을(를) 사용하여 오류 유형별로 별도의 조건 경로를 만들어 여정 오류 보고를 구체화합니다.
* 추적 및 감사를 위해 표현식에서 여정 버전 식별자, 인스턴스 식별자 및 샌드박스 이름 참조

**용어집:**

* **여정 속성**: 현재 여정 실행 인스턴스 *(제품별)에 대한 기술 메타데이터 필드를 포함하는 식 편집기의 범주*
* **instanceUID**: 지정된 프로필 실행 *(제품별)에 대한 여정 인스턴스의 고유 식별자*
* **lastErrorCode**: 여정에서 가장 최근에 실패한 활동의 오류 코드입니다. 가능한 값은 HTTP 코드, `capped`, `timedOut` 및 `error` *(제품별)*&#x200B;입니다.
* **lastNodeTypeInError**: 오류가 발생한 마지막 활동의 형식이며 이벤트, 흐름 제어 또는 작업 *(제품별)*&#x200B;일 수 있습니다.
* **externalKey**: 여정 인스턴스 *(제품별)을(를) 트리거한 개별 식별자(예: 프로필 ID)*

**보호 기능:**

* 여정 속성 필드 값은 실행 시 라이브 여정에서 직접 검색됩니다. 실행 전 유효성 검사에는 사용할 수 없습니다
* `lastErrorCode` 필드에는 HTTP 오류 코드, `capped`, `timedOut` 및 `error`과 같은 미리 정의된 값이 사용됩니다.
* 여정 속성 은 단순 및 고급 표현식 편집기의 여정 속성 범주 아래에서 사용할 수 있습니다

**용어:**

* 정식 이름: 여정 속성 — 약어: 없음 — 변형: 여정 기술 필드, 여정 메타데이터 필드
* 동의어: &quot;여정 속성&quot; = &quot;여정 기술 필드&quot;; &quot;instanceUID&quot; = &quot;여정 인스턴스 식별자&quot;
* 혼동하지 않음: journeyUID (여정 정의 식별) ≠ instanceUID (특정 프로필의 여정 실행 식별)

**FAQ:**

* **Q: 식 편집기에서 여정 속성 필드는 어디에서 찾을 수 있습니까?** — 이벤트 및 데이터 소스 아래의 여정 속성 범주 아래에 있는 단순 표현식 편집기와 고급 표현식 편집기 모두에 표시됩니다.
* **Q: 최대 가용량 규칙에 의해 삭제된 프로필을 어떻게 기록할 수 있습니까?** — `lastErrorCode == "capped"`에서 오류 경로 조건 필터링을 추가하고 사용자 지정 작업을 통해 이러한 프로필을 서드파티 시스템으로 푸시합니다.
* **Q: `journeyUID`과(와) `instanceUID`의 차이점은 무엇입니까?** — `journeyUID`은(는) 여정 정의를 식별하며, `instanceUID`은(는) 지정된 프로필의 특정 실행 인스턴스를 식별합니다.
* **Q: 예기치 않은 시스템 오류에 대해 어떤 오류 코드가 반환됩니까?** — 예기치 않은 오류의 기본값으로 사용되며 거의 발생하지 않아야 하는 `error` 코드입니다.
* **Q: 여정 속성 필드를 사용하여 작업 실패에 대한 Slack 경고를 보낼 수 있습니까?** — 예. 사용자 지정 작업에서 `lastNodeNameInError` 및 `lastErrorCode`을(를) 참조하여 Slack 알림에 오류 세부 정보를 포함합니다.
