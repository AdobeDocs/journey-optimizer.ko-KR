# 여정의 태그 관리 {#journey_tags}

Journey Optimizer 전문가는 태그를 사용하여 여정을 구성할 수 있습니다. 태그는 검색을 개선하기 위해 개체를 빠르고 쉽게 분류하는 방법입니다.

## 태그와 이름 지정 규칙 비교 {#tags-vs-naming}

팀은 종종 복잡한 이름 지정 규칙을 사용하여 메타데이터를 여정 이름에 직접 저장합니다(예: *라이프사이클 마케팅 - 교육 - 고객 온보딩 V2 - 앱 교육 - 2025년 3분기*). 선의에서 이 접근 방식은 팀 구성원 간에 작업이 확장됨에 따라 규칙이 일관되게 적용되는 경우가 거의 없고 여정 목록을 탐색하기 어려워진다는 중요한 약점이 있습니다.

Journey Optimizer의 **태그 범주**&#x200B;은(는) 더 나은 대안을 제공합니다. 이름에서 메타데이터를 인코딩하는 대신, 분류된 태그를 각 여정(예: 팀, 목표, 단계, 분기)에 첨부하고 필터를 사용하여 찾습니다. 그런 다음 여정 이름은 실제 문제가 되는 것, 즉 추진되는 고객 이정표에 집중할 수 있습니다.

이름 지정 규칙보다 태그 범주의 이점:

* **일관성** — 태그는 자유롭게 입력하지 않고 제어된 목록에서 선택됩니다.
* **필터링 가능성** — 모든 태그 값 조합을 사용하여 여정 목록을 즉시 분할할 수 있습니다.
* **명확성** — 여정 이름은 짧고 이정표에 중점을 둡니다.
* **확장성** — 새 메타데이터 차원을 추가하는 것은 이름 지정 규칙을 다시 쓰지 않고 새 태그 범주를 만드는 것을 의미합니다.

권장 설정 워크플로에 대해서는 아래의 [여정 관리에 대한 태그 범주 설정](#tags-setup)을 참조하십시오.

## 여정에 태그 추가

여정 속성의 **태그** 필드를 사용하면 여정의 태그를 정의할 수 있습니다. 기존 태그를 선택하거나 새 태그를 만들 수 있습니다. 원하는 태그의 이름을 입력하고 목록에서 선택합니다. 사용할 수 없는 경우 **만들기**&#x200B;를 클릭하여 새 항목을 만들어 여정에 추가하십시오. 태그 정의 수에는 제한이 없습니다.

정의한 태그 목록은 **태그** 필드 아래에 표시됩니다.

>[!NOTE]
>
> 태그는 대/소문자를 구분하지 않습니다.
> 
> 여정을 복제하거나 새 버전을 만드는 경우 태그가 유지됩니다.

## 태그로 필터링

여정 목록에는 태그를 쉽게 시각화할 수 있도록 전용 열이 표시됩니다.

여정 특정 태그가 있는 태그만 표시하는 필터도 사용할 수 있습니다.

![여정 분류에 사용 가능한 태그가 있는 태그 선택 드롭다운](assets/tags2.png)

모든 유형의 여정(라이브, 초안 등)에서 태그를 추가하거나 제거할 수 있습니다. 여정 옆에 있는 **추가 작업** 아이콘을 클릭하고 **태그 편집**&#x200B;을 선택합니다.

![분류된 여정을 표시하는 태그로 필터링된 여정 목록](assets/tags3.png)

## 태그 관리

관리자는 **관리** 아래의 **태그** 메뉴를 사용하여 태그를 삭제하고 카테고리별로 정리할 수 있습니다. 이 [설명서](https://experienceleague.adobe.com/docs/experience-platform/administrative-tags/overview.html?lang=ko)를 참조하세요.

>[!NOTE]
>
> 여정에 정의된 태그가 기본 제공 &quot;분류되지 않음&quot; 카테고리에 추가됩니다.

## 여정 관리에 사용할 태그 범주 설정 {#tags-setup}

복잡한 명명 규칙을 팀 전체에서 태그 기반 접근 방식으로 바꾸려면 다음 단계를 따르십시오.

**1단계 — 태그 범주 만들기(관리자)**

**[!UICONTROL 관리]** > **[!UICONTROL 태그]**&#x200B;에서 팀이 현재 여정 이름으로 인코딩하는 각 메타데이터 특성에 대해 범주를 하나 만듭니다. 예: *팀*, *마케팅 목표*, *캠페인*, *단계*, *분기*.

**2단계 — 각 범주를 태그 값으로 채우기(관리자)**

각 카테고리 내에서 가능한 모든 값을 나타내는 태그를 만듭니다. 예를 들어, *단계* 범주에는 *인식*, *온보딩*, *유지*, *되먹임*&#x200B;이 포함될 수 있습니다.

**3단계 — 여정을 만들 때 태그를 적용합니다(실무자)**

새 여정이 만들어질 때마다 여정 속성의 각 카테고리에서 적절한 태그를 선택합니다. 여정은 일반적으로 카테고리당 하나의 태그를 전달합니다.

**4단계 — 마일스톤의 이름 여정, 태그로 필터링**

여정 이름을 이 이름이 유도하는 고객 마일스톤(예: *첫 번째 충성도 거래*)에 초점을 맞추십시오. 여정 목록의 여정 필터를 사용하여 이름 구문 분석에 의존하지 않고 원하는 메타데이터 조합으로 태그를 찾습니다.
