# 여정 유형 및 선택 안내서 {#journey-types-selection}

Adobe Journey Optimizer은 서로 다른 시작 메커니즘 및 비즈니스 시나리오에 맞게 디자인된 4가지 여정 유형을 지원합니다. 이 안내서는 차이점을 이해하고 사용 사례에 적합한 유형을 선택하는 데 도움이 됩니다.

## 여정 유형 개요 {#journey-types}

>[!BEGINTABS]

>[!TAB 단일 여정]

**사용 시기:** 실시간, 이벤트 트리거 경험

**단일 여정**&#x200B;은(는) 특정 작업(구매, 앱 로그인, 양식 제출)이 발생할 때 개별적으로 트리거됩니다. 프로필은 실시간으로 한 번에 하나씩 입력하므로 즉각적인 비헤이비어 기반 응답에 이상적입니다.

**완벽한 대상:** 구매 후 주문 확인, 누군가 구독할 때 환영 이메일, 탐색으로 트리거된 장바구니 포기, 암호 재설정 알림.

➡️ [이벤트에 대해 알아보기](../event/about-events.md) | [구독자에게 보내는 메시지 사용 사례](message-to-subscribers-uc.md)

>[!TAB 대상 여정 읽기]

**사용할 시기:** 대상자 세그먼트에 예약된 캠페인

**대상 여정 읽기** Adobe Experience Platform 대상으로 시작하여 모든 프로필에 동시에 메시지를 일괄적으로 보냅니다. 이 여정 유형은 예약된 대규모 통신에 이상적입니다.

**완벽한 대상:** 월간 뉴스레터, 대상 세그먼트에 대한 프로모션 캠페인, 제품 공지 및 시즌 마케팅 캠페인

➡️ [대상자 읽기에 대해 알아보기](read-audience.md) | [대상자 시작](../audience/about-audiences.md)

>[!TAB 대상 자격 여정]

**사용할 시기:** 대상 멤버십 변경에 대한 실시간 응답

프로필이 특정 대상에 대해 자격이 되거나 종료되는 경우 **대상 자격 여정**&#x200B;이 트리거됩니다. 프로필은 실시간으로 기준을 충족하면 개별적으로 입력되므로 고객 행동이 변경될 때 즉시 참여할 수 있습니다.

**완벽한 대상:** VIP 계층 업그레이드 알림, 고객이 비활성화될 때 다시 참여, 첫 번째 구매 축하 메시지 및 고객이 이동할 때 지리적 타깃팅.

➡️ [대상 자격에 대해 알아보기](audience-qualification-events.md) | [대상자 만들기](../audience/creating-a-segment-definition.md)

>[!TAB 비즈니스 이벤트 여정]

**사용할 시기:** 여러 고객에게 영향을 주는 비즈니스 조건

**비즈니스 이벤트 여정**&#x200B;은(는) 여러 프로필에 동시에 영향을 주는 비즈니스 수준 이벤트(재고 업데이트, 날씨 알림, 가격 변경)에 의해 트리거됩니다. 이들은 개별적인 행위보다는 더 넓은 비즈니스 조건에 반응한다.

**완벽한 대상:** 관심 있는 고객에 대한 재고 부족 알림, 플래시 판매 알림, 날씨 기반 프로모션, 가격 하락 알림 및 재고 부족 알림.

➡️ [비즈니스 이벤트에 대해 알아보기](../event/about-creating-business.md) | [항목 관리](entry-management.md)

>[!ENDTABS]

## 의사 결정 안내서: 여정 유형 선택 {#decision-guide}

이 진단트리에 따라 사용 사례에 적합한 여정 유형을 선택합니다.

### 1단계: 여정을 트리거하는 것은 무엇입니까?

* **고객이 특정 작업을 수행**(구매, 클릭, 로그인) → 2단계로 이동
* **시간/일정**(특정 시간 또는 반복 전송) → **대상 읽기 여정 사용**
* **고객 상태 변경**(세그먼트 가입/탈퇴) → 3단계로 이동
* **비즈니스 상태**(재고 수준, 가격 변경, 날씨) → **비즈니스 이벤트 여정 사용**

### 2단계: 개별 고객 조치 트리거

* **즉각적인 실시간 응답이 필요합니까?**
   * 예 → **단일 여정 사용**
   * 예약된 →으로 대상자 읽기 여정 고려 안 함

### 3단계: 고객 상태 변경

* **고객이 세그먼트를 입력하거나 종료할 때 응답해야 합니까?**
   * 예 → **대상 자격 여정 사용**
   * 아니요, 이벤트를 사용하여 단일 여정 고려 또→ 대상 필터를 사용하여 대상 읽기 를 입력하는 경우에만 해당합니다.

### 사용 사례별 빠른 선택

| 목표 | 권장 여정 유형 | 이유 |
|-----------|------------------------|-----|
| 구매 후 주문 확인 보내기 | 단일 | 개별 작업에 대한 즉각적인 응답 |
| 구독자에게 월별 뉴스레터 전송 | 대상자 읽기 | 예약된 일괄 통신 |
| 고객이 VIP 상태에 도달하면 알림 | 대상자 선별 | 상태 변경에 대한 실시간 응답 |
| 시청한 항목에 대한 재고가 적다는 것을 고객에게 알림 | 비즈니스 이벤트 | 비즈니스 상황은 여러 고객에게 영향을 미칩니다 |
| 새 앱 사용자 시작 | 단일 | 등록 이벤트에 의해 트리거됨 |
| 비활성 고객 재참여 | 대상자 선별 | 비활성 세그먼트 항목에 응답합니다 |
| 대상 세그먼트에 대한 시즌 프로모션 | 대상자 읽기 | 대상에 대해 예약된 캠페인 |
| Flash 판매 발표 | 비즈니스 이벤트 | 비즈니스 결정은 여러 고객에게 영향을 미칩니다. |

>[!NOTE]
>
>어떤 유형을 선택할지 확실하지 않습니까? 이벤트 기반 여정의 경우 **단일 여정**, 예약된 캠페인의 경우 **대상 경험 읽기**&#x200B;로 시작하십시오. 이러한 이벤트는 가장 일반적인 사용 사례를 다룹니다.

## 여정 유형 세부 비교 {#journey-types-comparison}

이 표를 사용하여 여정 유형을 빠르게 비교하고 사용 사례에 적합한 유형을 선택합니다.

| Aspect | 단일 여정 | 대상 여정 읽기 | 대상 자격 여정 | 비즈니스 이벤트 여정 |
|--------|------------------|------------------------|--------------------------------|------------------------|
| **시작 메커니즘** | 개별 이벤트 트리거 | 예약된 일괄 처리 | 실시간 대상 멤버십 변경 | 비즈니스 수준 이벤트 |
| **시작 시간** | 이벤트 발생 시 실시간 | 예약됨(1회 또는 반복) | 검증 발생 시 실시간 | 비즈니스 이벤트 발생 시 실시간 |
| **프로필 항목** | 한 번에 하나씩 | 한 번에 모두(일괄 처리) | 한 번에 하나씩 | 여러 프로필 동시 실행 |
| **트리거 원본** | 고객 작업(구매, 클릭, 로그인) | 시간 기반 일정 | 대상자 멤버십(시작/종료) | 비즈니스 상태(재고, 날씨, 가격) |
| **추천 항목** | 트랜잭션 메시지, 행동 응답 | 마케팅 캠페인, 뉴스레터 | 충성도 프로그램, 라이프사이클 단계 | 재고 경고, 프로모션, 비즈니스 상태 |
| **다음의 경우에 사용** | 필요한 개별 작업에 대한 즉각적인 응답 | 일정에 따라 대규모 대상 세그먼트에 도달 | 고객 상태 변경에 대한 응답 | 여러 고객에게 영향을 미치는 비즈니스 이벤트 |
| **예** | 주문 확인, 암호 재설정 | 월간 뉴스레터, 시즌 캠페인 | VIP 업그레이드, 비활성 경고 | 낮은 재고 경고, 플래시 판매, 가격 하락 |
| **다시 시작** | 구성 가능(프로필당 여러 항목 허용) | 각 프로필은 실행당 한 번씩 입력됩니다 | 자격 이벤트별 구성 가능 | 여러 프로필이 동일한 이벤트의 영향을 받을 수 있음 |
| **데이터 요구 사항** | 트리거 데이터가 있는 이벤트 스키마 | Adobe Experience Platform 대상 | 스트리밍 또는 일괄 처리 대상 | 비즈니스 이벤트 스키마 |
