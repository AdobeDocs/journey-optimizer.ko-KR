# 제한 사항 {#journey-limitations}

>[!BEGINSHADEBOX]

**이 페이지에서:** 작업, 버전, 사용자 지정 작업, 이벤트 및 데이터 원본을 포함하여 여정에 적용되는 제한 사항과 보호 기능을 검토하십시오.

>[!ENDSHADEBOX]

다음은 여정 사용과 관련된 제한 사항입니다.

## 일반 작업 제한 사항 {#action-limitations}

* 전송 제한이 없습니다.
* 오류가 발생하면 시스템에서 세 번 다시 시도합니다. 수신된 오류 메시지에 따라 재시도 횟수를 조정할 수 없습니다.
* 기본 제공 **반응** 이벤트를 사용하면 기본 작업에 반응할 수 있습니다([페이지](../building-journeys/reaction-events.md) 참조). 사용자 지정 작업을 통해 보낸 메시지에 반응하려면 전용 이벤트를 구성해야 합니다.
* 두 가지 작업을 병렬로 배치할 수 없으며 하나씩 추가해야 합니다.


## 여정 버전 제한 사항 {#journey-versions-limitations}

* v1에서 이벤트 활동으로 시작하는 여정은 이후 버전에서 이벤트 이외의 다른 것으로 시작할 수 없습니다. **대상 자격** 이벤트로 여정을 시작할 수 없습니다.
* v1에서 **대상 자격** 활동으로 시작하는 여정은 이후 버전에서 항상 **대상 자격**(으)로 시작해야 합니다.
* **대상 자격**(첫 번째 노드)에서 선택한 대상 및 네임스페이스는 새 버전에서 변경할 수 없습니다.
* 재진입 규칙은 모든 여정 버전에서 동일해야 합니다.
* **대상자 읽기**&#x200B;로 시작하는 여정은 다음 버전에서 다른 이벤트로 시작할 수 없습니다.

## 사용자 지정 작업 제한 사항 {#custom-actions-limitations}

* 사용자 정의 작업 URL은 동적 매개 변수를 지원하지 않습니다. 
* POST 및 PUT 호출 메서드만 지원됩니다. 
* 쿼리 매개 변수 또는 헤더의 이름은 &quot;.&quot; 또는 &quot;$&quot;로 시작할 수 없습니다. 
* IP 주소는 허용되지 않습니다. 
* 내부 Adobe 주소(.adobe.) 허용되지 않습니다.

## 이벤트 제한 사항 {#events-limitations}

* 시스템 생성 이벤트의 경우 고유한 오케스트레이션 ID를 얻으려면 먼저 고객 여정을 시작하는 데 사용되는 스트리밍 데이터를 Journey Optimizer 내에서 구성해야 합니다.. 이 오케스트레이션 ID는 [!DNL Adobe Experience Platform]&#x200B;(으)로 들어오는 스트리밍 페이로드에 추가되어야 합니다. 이 제한은 규칙 기반 이벤트에는 적용되지 않습니다.

## 반응 이벤트 제한 사항 {#reaction-limitations}

* **[!UICONTROL 반응]** 활동은 여정 캔버스에서 [채널 작업 활동](../building-journeys/journey-action.md) 바로 뒤에 배치해야 합니다. 채널 액션과 **[!UICONTROL 반응]** 활동 사이에 **[!UICONTROL 대기]** 활동 또는 다른 활동을 배치하는 것은 지원되지 않으며 예상대로 반응이 작동하지 않을 수 있습니다. 자세한 내용은 [이 섹션](../building-journeys/reaction-events.md)을 참조하십시오.

## 데이터 소스 제한 사항 {#data-sources-limitations}

* 고객 여정 내에서 외부 데이터 소스를 활용하여 실시간으로 외부 데이터를 조회할 수 있습니다. 이러한 소스는 REST API를 통해 사용할 수 있어야 하고 JSON을 지원하고 요청 볼륨을 처리할 수 있어야 합니다.

## 프로필 만들기와 동시에 시작하는 여정 {#journeys-limitation-profile-creation}

[!DNL Adobe Experience Platform]에서 API 기반 프로필 만들기/업데이트와 관련된 지연이 있습니다. 지연 시간 측면에서 SLT(서비스 수준 목표)는 20,000RPS(초당 요청) 볼륨에서 95번째 백분위수 요청에 대해 수집에서 통합 프로필까지 1분 미만입니다.

프로필 만들기와 동시에 여정이 트리거되고 Profile Service에서 정보를 즉시 확인/검색하면 제대로 작동하지 않을 수 있습니다.

다음 두 가지 솔루션 중 하나를 선택할 수 있습니다.

* 첫 번째 이벤트 후에 대기 활동을 추가하여 [!DNL Adobe Experience Platform]에게 Profile Service에 수집을 수행하는 데 필요한 시간을 제공합니다.

* 프로필을 즉시 활용하지 않는 여정을 설정합니다. 예를 들어 여정이 계정 만들기를 확인하도록 디자인된 경우 경험 이벤트에는 첫 번째 확인 메시지(이름, 성, 이메일 주소 등)를 보내는 데 필요한 정보가 포함될 수 있습니다.

## 대상자 제한 사항 읽기 {#read-audiences-limitations}

* 스트리밍 대상자는 항상 최신 상태이지만 배치 대상자는 검색하는 순간에 계산되지 않습니다. 일일 배치 평가 시간에만 매일 평가됩니다.

**의도:**

* 여정 작업에 대한 전송 및 다시 시도 제한 이해
* 허용 또는 차단된 여정 버전 전환 알아보기
* 사용자 지정 작업 URL, 메서드 및 헤더 구성에 대한 제한 사항 식별
* 외부 시스템 통합을 위한 데이터 소스 요구 사항 이해
* 프로필 생성과 동시에 여정을 시작할 때 시간 문제 방지

**용어집:**

* **반응 이벤트**: 채널 작업(예: 이메일 열기 또는 클릭)과 프로필의 상호 작용을 수신하는 여정 활동으로, 채널 작업 활동 바로 뒤에 배치해야 합니다. *(제품별)*
* **규칙 기반 이벤트**: 트리거가 시스템 생성 오케스트레이션 ID가 아닌 논리 조건으로 정의되는 이벤트 유형입니다. *(제품별)*
* **SLT(서비스 수준 목표)**: Adobe Experience Platform에서 API 기반 프로필 만들기/업데이트에 대한 지연 벤치마크입니다. 20K RPS의 95번째 백분위수에서 수집에서 통합 프로필까지 1분 미만입니다.

**보호 기능:**

* 전송 제한이 적용되지 않습니다. 오류 시 세 번의 재시도가 자동으로 수행되며 조정할 수 없습니다
* 두 작업은 동시에 실행할 수 없으며 순차적으로 추가되어야 합니다
* v1의 여정 활동으로 시작하는 이벤트는 이후 버전에서 이벤트가 아닌 활동으로 시작할 수 없습니다
* v1에서 대상 자격으로 시작하는 여정은 모든 후속 버전에서 항상 대상 자격으로 시작해야 합니다. 대상 및 네임스페이스는 변경할 수 없습니다
* 대상자 읽기로 시작하는 여정은 다음 버전의 다른 이벤트로 시작할 수 없습니다
* 사용자 지정 작업 URL은 동적 매개 변수를 지원하지 않습니다. POST 및 PUT 호출 메서드만 지원됩니다
* 사용자 지정 작업 쿼리 매개 변수 및 헤더 이름은 &quot;.&quot;로 시작할 수 없습니다. 또는 &quot;$&quot;; IP 주소 및 내부 Adobe 주소(.adobe.) 허용되지 않음
* 반응 활동은 채널 작업 활동 직후에 배치해야 합니다. 이들 사이에 대기 또는 다른 활동을 삽입하는 것은 지원되지 않습니다
* 외부 데이터 소스는 REST API를 통해 액세스할 수 있어야 하며, JSON을 지원하고 요청 볼륨을 처리해야 합니다
* 배치 대상은 일일 배치 평가 시간에 하루에 한 번만 평가되며, 검색 시에는 다시 계산되지 않습니다
* 프로필 생성과 동시에 여정이 트리거되면 플랫폼 수집 지연으로 인해 프로필 데이터를 아직 사용할 수 없습니다

**용어:**

* 정식 이름: 여정 제한 — 약어: 없음 — 변형: 여정 보호, 여정 제한
* 혼동하지 마십시오. &quot;반응 이벤트 제한&quot; ≠ &quot;일반 작업 제한&quot; — 반응 이벤트는 채널 작업 바로 뒤에 배치해야 합니다. 일반 작업 제한은 재시도, 병렬 처리 및 제한을 다룹니다

**FAQ:**

* **Q: Journey Optimizer에서 실패한 작업을 몇 번 다시 시도합니까?** — 세 번의 재시도가 자동으로 수행되므로 재시도 횟수를 구성할 수 없습니다.
* **Q: 채널 작업과 반응 이벤트 사이에 대기 활동을 배치할 수 있습니까?** — 아니요. 채널 작업 활동 바로 뒤에 반응 이벤트를 배치해야 합니다. 사이에 활동을 추가하는 것은 지원되지 않으며 반응 이벤트가 예상대로 작동하지 않을 수 있습니다.
* **Q: 새 여정 버전을 만들 때 첫 번째 이벤트 유형을 변경할 수 있습니까?** — 아니오. v1에 설정된 엔트리 메커니즘은 이후의 모든 버전에서 유지되어야 합니다. 이벤트로 시작하는 여정은 이벤트로 계속 시작해야 하며 대상 자격으로 시작하는 여정은 항상 대상 자격으로 시작해야 합니다.
* **Q: 프로필이 만들어지는 동시에 트리거될 때 여정이 작동하지 않는 이유는 무엇입니까?** — API를 통한 프로필 작성은 통합 프로필에서 데이터를 사용할 수 있기까지 지연 시간이 있습니다(SLT &lt; 1분, 95번째 백분위수). 첫 번째 이벤트 후에 대기 활동을 추가하면 Platform에서 수집을 완료할 수 있습니다.
* **Q: 스트리밍 대상이 항상 여정에서 최신 상태입니까?** — 예. 스트리밍 대상자는 항상 최신 상태입니다. 그러나 배치 대상은 검색 시점이 아니라 일별 배치 평가 시간에 하루에 한 번만 평가됩니다.
