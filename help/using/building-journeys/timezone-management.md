# 시간대 관리 {#timezone_management}

>[!BEGINSHADEBOX]

**이 페이지에서:** 여정의 시간대(고정 시간대 또는 각 프로필에서 가져온 시간대)를 설정하여 시간 조건, 날짜 조건 및 사용자 지정 대기 횟수와 같은 시간 기반 활동 실행 시기를 제어하는 방법에 대해 알아봅니다.

>[!ENDSHADEBOX]

>[!CONTEXTUALHELP]
>id="ajo_journey_properties_time_zone"
>title="여정 시간대"
>abstract="시간대 설정은 여정의 시간대를 정의합니다. 고정된 시간대를 사용하면 여정에 참여하는 모든 개인 사용자에게 동일하게 적용됩니다."


여정의 [속성](../building-journeys/journey-properties.md#timezone)에서 시간대를 정의할 수 있습니다.

여정 속성에 액세스하려면 화면 오른쪽 상단의 연필 아이콘을 선택합니다.

이 시간대는 다음과 같은 시간 요소를 포함하는 여정의 모든 활동에 사용됩니다.

* [시간 조건](../building-journeys/conditions.md#time_condition)
* [날짜 조건](../building-journeys/conditions.md#date_condition)
* [사용자 지정 대기](../building-journeys/wait-activity.md#custom)

[고정 시간대](#fixed-timezone)를 선택하거나 사용자 프로필에 정의된 시간대 [을(를) 사용하도록 선택할 수 있습니다](#timezone-from-profiles).

## 고정 시간대 정의 {#fixed-timezone}

시간대를 고정할 수 있습니다. 사전 정의된 시간대를 지우고 드롭다운 목록에서 시간대를 선택합니다. 고정된 시간대를 사용하는 경우 여정에 들어오는 모든 개인에 대해 동일합니다.

이렇게 하려면 **[!UICONTROL 여정 속성]** 창에서 표준 시간대를 선택하십시오.

![여정 속성의 시간대 선택 드롭다운](assets/journey72.png)

## 프로필 시간대 사용 {#timezone-from-profiles}

>[!CONTEXTUALHELP]
>id="ajo_journey_properties_profile_time_zone"
>title="프로필 시간대 사용"
>abstract="이 옵션은 **대기** 및 **조건** 활동에서 실시간 프로필 시간대를 사용합니다. 프로필에 대해 시간대가 정의된 경우 여정에서 해당 시간대를 검색하고 이를 사용합니다. 그렇지 않은 경우 시간대는 위의 시간대 필드에 정의된 시간대가 됩니다."

여정의 시작 이벤트에 네임스페이스가 있어 여정이 [!DNL Adobe Experience Platform]의 실시간 고객 프로필 서비스에 연결할 수 있는 경우 프로필 수준에서 정의된 시간대를 사용할 수 있습니다. 이렇게 하려면 **속성**&#x200B;에서 **대기 및 조건에 프로필 시간대 사용**&#x200B;을 확인하세요. 이 옵션은 기본적으로 선택되어 있지 않습니다.

여정에 대해 시간대가 정의된 경우 시간대를 검색하여 프로필에서 사용합니다. 시간대가 정의되어 있지 않으면 시간대 필드에 정의된 시간대가 사용됩니다.

![개인화된 타이밍에 대한 데이터 원본의 프로필 시간대 구성](assets/journey73.png)

>[!NOTE]
>
>프로필 시간대는 **환경 설정 세부 정보** 필드 그룹에 있는 **표준 시간대** 필드에서 작동합니다.

## 표현식에 시간대 사용 {#timezone-in-expressions}

여정의 시작 및 종료 날짜를 특정 시간대에 연결할 수 없습니다. 인스턴스의 시간대에 자동으로 연결됩니다.

이해를 돕기 위해 이 정보를 이 페이지의 설명서와 통합해야 합니다. 두 소스 모두 독립적으로 사용하기 위한 것은 아닙니다. 이 페이지에서는 기능에 대해 설명하지만, 용어, 의도, 적용 가능성 및 제약 조건을 명확히 하는 데 도움이 되는 추가 컨텍스트를 제공합니다.

* **TL;DR:** 이 페이지에서는 모든 프로필에 적용된 고정 시간대와 실시간 고객 프로필에서 가져온 프로필별 시간대 중 선택하여 Adobe Journey Optimizer 여정 속성에서 시간대 설정을 구성하는 방법에 대해 설명합니다.

**의도:**
* 모든 프로필이 조건 및 대기에 대해 동일한 시간 참조를 따르도록 여정에 고정 시간대를 설정합니다
* 대기 및 조건 활동이 각 개인의 저장된 시간대 기본 설정을 사용하도록 프로필별 시간대 활성화
* 여정 시간대 설정의 영향을 받는 여정 활동 이해
* 개별 시간대 값을 저장하는 프로필 필드 그룹을 식별합니다

**용어집:**
* **고정 시간대**: *(제품별) 여정을 입력하는 모든 프로필에 균일하게 적용되는 여정 속성에서 선택한 단일 시간대*
* **프로필 시간대**: &quot;대기 및 조건에 프로필 시간대 사용&quot; 옵션이 활성화된 경우 사용되는 기본 설정 세부 정보 필드 그룹의 `timeZone` 필드에 저장된 개인별 시간대 *(제품별)*
* **환경 설정 세부 정보 필드 그룹**: 프로필 수준 시간대 확인에 사용되는 `timeZone` 특성이 포함된 XDM 필드 그룹입니다.

**보호 기능:**
* &quot;대기 및 조건에서 프로필 시간대 사용&quot; 옵션은 여정의 시작 이벤트에 네임스페이스가 있는 경우(즉, 여정이 실시간 고객 프로필 서비스에 연결할 수 있는 경우)에만 사용할 수 있습니다
* 옵션은 기본적으로 선택되어 있지 않습니다. 명시적으로 활성화하지 않는 한 고정 시간대가 사용됩니다.
* 옵션을 활성화했지만 프로필에 정의된 시간대가 없는 경우 여정은 여정 속성에 정의된 고정 시간대로 돌아갑니다
* 여정 시작 및 종료 날짜는 특정 시간대에 연결할 수 없으며 인스턴스의 시간대와 자동으로 연결됩니다

**용어:**
* 정식 이름: 시간대 관리 — 약어: 없음 — 변형: 시간대 구성, 여정 시간대
* 동의어: &quot;고정 시간대&quot; = &quot;모든 개인에 대해 동일&quot;; &quot;프로필 시간대&quot; = &quot;대기 및 조건에 프로필 시간대 사용&quot;
* 혼동하지 마십시오. &quot;여정 시간대&quot;(활동에 적용) ≠ &quot;인스턴스 시간대&quot;(여정 시작/종료 날짜에 적용, 자동으로 설정)

**FAQ:**
* **Q: 여정의 표준 시간대를 어디에 설정해야 합니까?** — 여정 속성 창에서 여정 캔버스의 오른쪽 상단에 있는 연필 아이콘을 통해 액세스할 수 있습니다.
* **Q: 어떤 활동이 여정 시간대를 사용합니까?** — 시간 조건, 날짜 조건 및 사용자 지정 대기 활동.
* **Q: 각 프로필이 고유한 현지 시간대를 따르도록 하려면 어떻게 해야 합니까?** — 여정 속성에서 &quot;대기 및 조건에 프로필 시간대 사용&quot; 옵션을 활성화합니다. 이렇게 하려면 여정에 실시간 고객 프로필 서비스에 연결할 수 있는 네임스페이스가 있어야 합니다.
* **Q: 프로필에 정의된 시간대가 없고 프로필 시간대 옵션이 활성화되어 있으면 어떻게 됩니까?** — 여정이 여정 속성의 시간대 필드에 정의된 고정 시간대로 폴백됩니다.
* **Q: 개인의 시간대를 저장하는 프로필 필드는 무엇입니까?** — 프로필 스키마의 환경 설정 세부 정보 필드 그룹 내의 `timeZone` 필드입니다.
* **Q: 여정의 시작 날짜와 종료 날짜를 특정 시간대로 설정할 수 있습니까?** — 아니요. 여정 시작 및 종료 날짜는 인스턴스의 시간대와 자동으로 연결되므로 사용자 지정 시간대에 연결할 수 없습니다.
