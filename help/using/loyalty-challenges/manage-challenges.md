## 과제 라이프사이클 {#challenge-lifecycle}

과제는 다양한 상태로 이동합니다.

* **초안**: 만들거나 편집하는 중
* **예약됨**: 게시됨, 시작 날짜에 활성화됨
* **Live**: 활성 상태이며 고객이 참여할 수 있습니다.
* **완료됨**: 합격 종료 날짜 또는 목표 충족됨
* **중지됨**: 완료하기 전에 수동으로 중지됨
* **보관됨**: 조직용으로 보관됨

## 문제 편집 {#edit-challenges}

현재 상태에 따라 문제를 편집할 수 있습니다.

* **초안 문제**: 전체 편집 기능
* **예약된/라이브 문제**: 제한된 편집(콘텐츠 업데이트, 날짜 확장)

주요 수정이 필요한 변경 사항의 경우 문제를 복제하고 새 버전을 만드십시오.

## 과제 복제 {#duplicate-challenges}

다음과 같은 과제의 중복:
* 새로운 기간에 대해 성공한 과제 재실행
* 다양한 대상을 위한 변형 만들기
* 작업 요구 사항 또는 보상 업데이트
* 중지되거나 완료된 문제 다시 활성화

## 성능 모니터링 {#monitor-performance}

다음을 통해 과제 성과 추적:

* **기여도 지표**: 등록, 활성 참가자
* **완료 지표**: 완료율, 평균 완료 시간
* **보상 지표**: 배포된 총 보상, 유형별 보상
* **참여 지표**: 콘텐츠 카드 노출 횟수, 메시지 게재

과제 성능 탭의 성능 데이터 및 자동 생성된 여정 보고서에 액세스합니다.

## 모범 사례 {#best-practices}

### 과제 만들기

* 간단한 첫 번째 과제 시작
* 프로덕션에 게시하기 전에 철저하게 테스트
* 요구 사항 및 보상에 대한 명확한 커뮤니케이션 보장
* 고객이 과제를 완료할 수 있도록 현실적인 타이밍 설정

### 관리 및 모니터링

* 과제 성과를 정기적으로 점검(라이브 과제의 경우 최소 매주)
* 목적, 대상자 또는 기간을 나타내는 수사적 이름 사용
* 조직에 일관된 태그 적용
* 완료된 문제를 아카이빙하여 재고를 관리

## 문제 해결 {#troubleshooting}

일반적인 문제 및 솔루션:

* **문제가 고객에게 표시되지 않음**: 상태, 대상자 자격, 콘텐츠 카드 구성 확인
* **낮은 참여율**: 콘텐츠 가시성, 메시징 명확성, 작업 달성 가능성 검토
* **작업이 트리거되지 않음**: 데이터 수집, 이벤트 특성, 대상 자격 요건 확인
* **보상 할당되지 않음**: 구성 및 외부 시스템 연결 확인
