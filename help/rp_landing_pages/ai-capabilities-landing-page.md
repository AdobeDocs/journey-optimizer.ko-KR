# Adobe Journey Optimizer의 AI 기능{#section-overview}

Adobe Journey Optimizer은 인공 지능과 머신 러닝의 힘을 활용하여 고객 경험을 만들고, 최적화하고, 전달하는 방법을 혁신합니다. AI 기능은 채널 전반에서 개인화된 콘텐츠를 생성하는 것부터 최적의 고객 참여 시간을 예측하는 것까지 워크플로를 간소화하고 효과를 극대화합니다. 이 섹션에서는 AI 기반 기능이 함께 작동하여 보다 현명한 결정을 내리고, 복잡한 작업을 자동화하고, 대상자에게 진정한 반향을 불러일으키는 경험을 만드는 방법에 대해 살펴봅니다. 콘텐츠 생성을 위해 생성 AI를 활용하든, 의사 결정을 위해 예측 모델을 사용하든, 더 나은 참여를 위해 전송 시간을 최적화하든 관계없이 고객 여정 오케스트레이션에서 AI의 모든 잠재력을 발휘할 수 있는 실용적인 도구와 전략을 발견할 수 있습니다.

- **[브랜드 정렬 점수](../using/content-management/brands-score.md)** - AI 기반 점수를 사용하여 AI 생성 콘텐츠가 브랜드 지침에 얼마나 잘 부합하는지 평가합니다.
- **[Experiment Accelerator](../using/content-management/experiment-accelerator-gs.md)** - AI 기반 인사이트 및 권장 사항을 통해 콘텐츠 실험 프로세스를 가속화합니다.

## 자주 묻는 질문

+++**AI 기능을 사용하려면 어떤 권한이 필요합니까?**

콘텐츠 생성에 AI 도우미를 사용하려면 사용자에게 **콘텐츠 생성** 권한이 부여되어야 합니다. 이 권한은 권한 제품의 AI Assistant 리소스를 통해 할당됩니다. 제품 지식 및 운영 통찰력에 AI Assistant를 사용하려면 사용자가 Adobe Experience Cloud 생성 AI 사용자 지침에 동의해야 합니다.

[권한에 대해 자세히 알아보기](../using/administration/ootb-permissions.md)

+++

+++**AI Assistant 콘텐츠 생성을 지원하는 채널은 무엇입니까?**

콘텐츠 생성을 위한 AI 도우미는 **이메일**, **푸시**, **SMS** 및 **웹** 채널에 사용할 수 있습니다. 현재 DM, 콘텐츠 카드, LINE 또는 WhatsApp 채널에는 사용할 수 없습니다.

+++

+++**AI Assistant를 사용하는 모범 사례는 무엇입니까?**

- **잘 정의된 프롬프트 사용** - 생성된 콘텐츠의 품질은 마케팅 목표와 프롬프트의 영향을 크게 받습니다. 구체적이고 명확해야 합니다.
- **브랜드 자산 업로드** - 정확한 브랜드 내 콘텐츠를 위해 PDF, JPEG, PNG 또는 ZIP 형식(최대 50MB)의 브랜드 자산을 제공합니다.
- **사용자 지정 템플릿 사용** - 최대 8~10개의 이미지와 함께 브랜드별 이메일 템플릿을 활용하여 최적의 결과를 얻을 수 있습니다.
- **피드백 제공** - 모델을 구체화하는 데 도움이 되는 엄지 손가락 위로, 아래로 또는 플래그 아이콘을 사용하여 문제가 있는 출력을 보고합니다.
- **세대당 하나의 브랜드 자산만 활용** - 여러 자산을 업로드할 수 있지만 각 세대에 하나만 사용하십시오.

[AI Assistant 가드레일에 대해 자세히 알아보기](../using/content-management/gs-generative.md#generative-guardrails)

+++

+++**전송 시간 최적화에 대한 모범 사례는 무엇입니까?**

- **30일 대기** - 전송 시간 최적화를 활성화하여 충분한 데이터 수집을 보장하려면 적어도 30일 동안 전자 메일 및 푸시 작업을 사용하십시오.
- **최적의 대기 시간을 선택하십시오** - 최상의 결과를 얻으려면 6-24시간 사이의 최대 대기 시간을 설정하십시오. 기간이 짧으면 최적화 가능성이 제한됩니다. 기간이 길면 메시지가 오래된 것일 수 있습니다.
- **올바른 지표에 맞게 최적화** - 전자 메일의 경우 동작을 실행할 때 클릭 수에 맞게 최적화하거나 정보 콘텐츠를 위해 엽니다. 푸시 메시지는 항상 [열기]에 최적화되어 있습니다.
- **시간에 민감한 메시지는 사용하지 않음** - 주문 확인, 암호 재설정 또는 비행 알림과 같은 긴급한 운영 메시지에는 전송 시간 최적화를 사용하지 마십시오. 프로모션 및 뉴스레터와 같은 마케팅 커뮤니케이션에 가장 적합합니다.
- **푸시할 아침 전송 예약** - 야간 알림을 방지하려면 아침마다 일괄 푸시 전송을 더 짧은 기간으로 예약하십시오(예: 오전 9시에 8시간 대기 전송).

[전송 시간 최적화에 대해 자세히 알아보기](../using/building-journeys/send-time-optimization.md)

+++

+++**전송 시간 최적화의 제한 사항은 무엇입니까?**

- **채널** - 여정의 이메일 및 푸시 알림 채널에만 사용할 수 있습니다. 캠페인이나 사용자 지정 작업을 통해 사용할 수 없습니다.
- **가용성** - Adobe 고객 지원 센터 또는 Adobe 담당자가 활성화해야 합니다.
- **교육 기간** - 사용하기 전에 최소 30일 동안의 내역 전자 메일 또는 푸시 데이터가 필요합니다.
- **모델 교육** - 모델은 처음에 지난 16주 동안의 데이터를 사용하여 매주 교육된 다음 매월 다시 교육됩니다.
- **탐색과 최적화 비교** - 메시지의 5%는 무작위 탐색 전송 시간을 받고, 95%는 최적화된 전송 시간을 받습니다.

+++

+++**의사 결정 시 AI 모델의 제한 사항은 무엇입니까?**

- **최대 AI 모델** - 조직당 최대 5개의 AI 등급 모델.
- **데이터 세트 요구 사항** - 자동 최적화 모델을 위해 지난 14일 내에 최소 2개의 오퍼에 100개 이상의 표시 이벤트와 5개 이상의 클릭 이벤트가 있어야 합니다.
- **피드백 이벤트** - 경험 이벤트로 전송해야 하며 Journey Optimizer 채널에서 자동으로 수집되지 않습니다.
- **API 제한 사항** - 자동 최적화 모델은 Batch Decisioning API에서 작동하지 않습니다(의사 결정 관리만 해당).

[AI 모델에 대해 자세히 알아보기](../using/experience-decisioning/ranking/ai-models.md)

+++

+++**AI 기반 의사 결정에 적용되는 보호 기능은 무엇입니까?**

| 구성 요소 | 제한 |
|-----------|-------|
| AI 등급 모델 | 조직당 5개 |
| 의사 결정 요청(Edge 세그먼테이션을 사용한 코드 기반) | 1,500/초 |
| 의사 결정 요청(Edge 세그먼테이션을 사용하지 않는 코드 기반) | 5,000/초 |
| 항목 컬렉션 | 총 10,000개 |
| 컬렉션당 오퍼 항목 | 500 |
| 의사 결정 정책별 선택 전략 | 10 |
| 결정 정책별로 반환된 오퍼 항목 | 30 |
| 자격 규칙 + 등급 공식 | 10,000 결합 |
| 규칙당 프로필 속성 | 25 |
| 규칙당 컨텍스트 데이터 속성 | 30 |

[Decisioning 보호 기능에 대해 자세히 알아보기](../using/experience-decisioning/decisioning-guardrails.md)

+++

+++**AI 기능을 사용하려면 약관에 동의해야 합니까?**

예. Journey Optimizer에서 AI Assistant를 사용하기 전에 [Adobe Experience Cloud Generative AI 사용 지침](https://www.adobe.com/kr/legal/licenses-terms/adobe-dx-gen-ai-user-guidelines.html)에 동의해야 합니다. 자세한 내용은 Adobe 담당자에게 문의하십시오. 또한 Adobe은 생성 AI 사용의 투명성을 위한 노력의 일환으로 Content Credentials을 Firefly 생성 에셋에 적용합니다.

+++

+++**AI 생성 컨텐츠가 항상 정확합니까?**

아니요. 생성 AI 콘텐츠가 항상 정확하지는 않을 수 있습니다. 항상 AI가 생성한 출력의 정확성을 검토하고 사용 사례에 적합한지 확인하십시오. 제공된 평가 도구를 사용하여 피드백을 공유하여 엔지니어가 모델을 구체화할 수 있습니다.
