# 테스트, 유효성 검사 및 승인{#section-overview}

이 섹션에서는 Journey Optimizer의 모든 테스트 및 승인 기능을 다룹니다. 테스트 프로필을 사용하여 콘텐츠를 미리 보고, 여정 논리의 유효성을 검사하며, 이메일 렌더링 및 스팸 점수를 확인하고, A/B 실험을 실행하며, 충돌을 감지하고, 승인 워크플로를 설정하는 도구를 찾을 수 있습니다.

이 랜딩 페이지에서는 구축하려는 대상(캠페인 또는 여정)에 따라 적합한 테스트 접근 방식을 선택할 수 있으며, 권장 테스트 워크플로를 안내하며, 모든 테스트 및 승인 리소스에 빠르게 액세스할 수 있습니다. 아래 [테스트 접근 방식 선택](#choose-your-testing-approach)을 시작으로 사용 사례에 적용되는 도구를 확인하세요. 주요 테스트 용어에 대한 정의는 [주요 용어](#key-terminology)를 참조하세요.

## 테스트 및 승인이 중요한 이유

테스트 및 승인 프로세스는 브랜드 평판을 보호하고 캠페인 성공을 보장하는 필수적인 품질 관리 단계 역할을 합니다. 중요한 이유는 다음과 같습니다.

* **고객에게 도달하기 전에 오류 포착** - 제어된 환경에서 깨진 링크, 잘못된 개인화, 렌더링 문제, 논리적 오류를 식별하여 신속하고 위험 부담 없이 수정할 수 있습니다.

* **전달성 향상** - 스팸 점수를 테스트하고, 이메일 인증의 유효성을 검사하며, 다양한 이메일 클라이언트에서 제대로 표시되는지 확인하여 받은 편지함 도달률과 참여율을 극대화합니다.

* **브랜드 일관성 보장** - 다양한 테스트 프로필을 사용하여 콘텐츠를 미리 보고, 개인화된 콘텐츠가 다양한 고객 세그먼트에 대해 올바르게 표시되며, 브랜드 표준을 유지하는지 확인합니다.

* **복잡한 여정 유효성 검사** - 여러 단계로 이루어진 고객 여정 흐름을 시뮬레이션하여 트리거가 올바르게 실행되고, 조건이 제대로 평가되며, 고객이 적시에 올바른 메시지를 수신하는지 확인합니다.

* **신뢰성 설정** - 이해 당사자의 서명이 필요한 공식 승인 워크플로를 구현하여 명확한 소유권을 확보하고 승인되지 않거나 시기상조인 캠페인 출시를 줄입니다.

* **시간 및 리소스 절약** - 개발 주기 초기에 문제를 발견하면 수정 비용이 저렴하고 신속하게 문제를 해결함으로써 출시 후 발생하는 비용이 많이 드는 수정이나 고객 서비스 에스컬레이션을 방지합니다.

## 테스트 접근 방식 선택

적합한 테스트 접근 방식은 구축할 콘텐츠와 유효성을 검사해야 할 콘텐츠에 따라 달라집니다. 이 안내서를 활용하여 시나리오와 가장 관련성이 높은 테스트 도구를 파악합니다.

>[!BEGINTABS]

>[!TAB 캠페인 테스트]

**모든 캠페인의 경우:**

* [테스트 프로필](../using/content-management/test-profiles.md) 또는 [샘플 입력 데이터](../using/test-approve/simulate-sample-input.md)를 사용하여 콘텐츠 미리 보기 및 테스트
* 장치 및 클라이언트에서 [이메일 렌더링](../using/content-management/rendering.md) 확인(이메일 채널만 해당)
* [스팸 점수 확인](../using/content-management/spam-report.md) 실행(이메일 채널만)
* 다른 캠페인 및 여정과의 [충돌](../using/conflict-prioritization/conflicts.md) 검토
* 이해 당사자 모니터링을 위한 [시드 목록](../using/configuration/seed-lists.md) 설정(이메일 채널만 해당)
* 활성화 전에 [승인](../using/test-approve/gs-approval.md) 제출

**A/B 테스트 및 최적화의 경우:**

* 다양한 처리 방식을 테스트하고 성능을 측정하기 위해 [콘텐츠 실험](../using/content-management/get-started-experiment.md)을 생성합니다.

**API 트리거 캠페인의 경우:**

* [캠페인 시뮬레이션 API](https://developer.adobe.com/journey-optimizer-apis/references/simulations/){target-&quot;_blank&quot;}를 사용하여 교정본 작업을 프로그래밍 방식으로 트리거합니다.

>[!TAB 여정 테스트]

**모든 여정의 경우:**

* 프로필 진행 상황을 시뮬레이션하려면 [테스트 모드](../using/building-journeys/testing-the-journey.md)를 사용합니다(초안 여정만 해당, 네임스페이스 필요). 메시지를 보내지 않고 실행 경로를 분석하려면 [시험 실행](../using/building-journeys/journey-dry-run.md)을 사용합니다.
* [미리 보기 및 교정본](../using/content-management/preview-test.md)을 사용하여 개별 메시지 테스트
* 다른 여정 및 캠페인과의 [충돌](../using/conflict-prioritization/conflicts.md) 확인
* 게시 전에 [승인](../using/test-approve/gs-approval.md) 제출

**복잡한 여정의 경우:**

* 테스트 모드와 시험 실행을 함께 사용하여 분기 논리와 실행 경로를 철저히 검증합니다.
* 다양한 항목 조건 및 프로필 속성을 체계적으로 테스트합니다.

**참고:** 충돌 감지 및 여정 제한 기능은 단일 여정, 대상자 선별 및 대상자 읽기 여정에만 사용할 수 있습니다.

>[!TAB 개인화 테스트]

**콘텐츠를 구축하기 전:**

* [개인화 플레이그라운드](../using/personalization/personalize.md#playground)에서 실험하여 구문을 학습하고 샘플 데이터를 사용하여 표현식을 테스트합니다.

**콘텐츠 제작 중:**

* [테스트 프로필](../using/content-management/test-profiles.md)을 사용하여 미리 보기를 통해 개인화 설정이 올바르게 표시되는지 확인합니다.
* CSV/JSON 파일에서 가져온 [샘플 입력 데이터](../using/test-approve/simulate-sample-input.md)를 사용하여 여러 시나리오를 테스트합니다(최대 30개 변형 지원).

>[!ENDTABS]

## 모범 사례 테스트

테스트 효과를 극대화하려면 다음 권장 사항을 따르세요.

1. **초기에 자주 테스트** - 캠페인이 완전히 구축될 때까지 기다리지 마세요. 콘텐츠, 개인화 및 논리를 개발 과정에서 점진적으로 테스트합니다.

1. **실제 테스트 프로필 사용** - 타깃 대상자 세그먼트를 정확하게 나타내는 [테스트 프로필을 생성](../using/audience/creating-test-profiles.md)하고, 예외 상황과 다양한 개인화 시나리오를 포함합니다.

1. **장치 및 클라이언트에서 테스트** - 널리 사용되는 이메일 클라이언트(Gmail, Outlook, Apple Mail) 및 장치(데스크톱, 모바일, 태블릿)에서 [이메일 렌더링](../using/content-management/rendering.md)을 확인하여 일관된 표시를 보장합니다(이메일 채널만 해당).

1. **개인화 설정을 철저히 검증** - 속성 값이 다른 여러 [테스트 프로필](../using/content-management/test-profiles.md)을 사용하여 개인화 토큰이 올바르게 렌더링되고 대체 값이 작동하는지 테스트합니다. [개인화 플레이그라운드](../using/personalization/personalize.md#playground)를 사용하여 개인화 표현식을 실험하고 캠페인에 적용하기 전에 샘플 데이터로 코드를 테스트합니다.

1. **샘플 데이터를 사용하여 콘텐츠 변형 테스트** - CSV 또는 JSON 파일에서 가져온 [샘플 입력 데이터](../using/test-approve/simulate-sample-input.md)를 사용하면 여러 테스트 프로필을 만들지 않고도 최대 30개의 개인화 시나리오를 테스트할 수 있으므로 시간을 절약하면서 포괄적인 테스트 범위를 확보할 수 있습니다. 이메일, SMS, 푸시, 웹, 코드 기반 경험, 인앱 및 콘텐츠 카드 채널을 지원합니다.

1. **이해 당사자 모니터링에 시드 목록 사용** - 품질 모니터링 및 규정 준수 확인을 위해 실행 시점에 모든 결과물의 사본을 받을 내부 이해 당사자를 자동으로 포함하도록 [시드 목록](../using/configuration/seed-lists.md)을 구성합니다(이메일 채널만 해당).

1. **여정 경로 시뮬레이션** - 분기가 여러 개인 복잡한 여정의 경우 [테스트 모드](../using/building-journeys/testing-the-journey.md)를 사용하여 다양한 입력 조건과 프로필 속성을 테스트하여 가능한 모든 경로의 유효성을 검사합니다. 네임스페이스를 사용하는 초안 여정에 사용할 수 있습니다.

1. **전달성 지표 확인** - 대량 발송 전에 [스팸 점수](../using/content-management/spam-report.md), 인증 상태 및 이메일 상태 지표를 검토합니다(이메일 채널만 해당).

1. **테스트 결과 문서화** - 향후 테스트 프로세스를 개선하고 팀과 학습 내용을 공유하기 위해 테스트 결과, 발견된 문제, 해결 방법을 기록합니다.

1. **초기 단계부터 이해 당사자 참여** - 피드백을 수집하고 기대치를 조정하기 위해 [공식 승인](../using/test-approve/gs-approval.md) 전에 이해 당사자와 미리 보기 및 테스트 결과를 공유합니다.

## 권장 테스트 워크플로

출시 전에 캠페인과 여정의 유효성을 검사하려면 다음 4단계 접근 방식을 따르세요.

| 단계 | 테스트 항목 | 주요 액션 |
|-------|-------------|-------------|
| **1. 콘텐츠 유효성 검사** | 개인화, 디자인, 렌더링 | [테스트 프로필을 사용한 미리 보기](../using/content-management/preview-test.md), CSV/JSON을 사용한 [여러 변형](../using/test-approve/simulate-sample-input.md) 테스트, 장치 간 [렌더링](../using/content-management/rendering.md) 확인 |
| **2. 기술 검사** | 전달성, 링크, 충돌 | [스팸 점수 확인](../using/content-management/spam-report.md) 실행, 링크 유효성 검사, 다른 캠페인과의 [충돌](../using/conflict-prioritization/conflicts.md) 확인 |
| **3. 여정 논리**(여정 전용) | 입력 조건, 흐름, 분기 | 진행 상황을 시뮬레이션하려면 [테스트 모드](../using/building-journeys/testing-the-journey.md)를 사용하고, 복잡한 경로의 경우 [드라이 런](../using/building-journeys/journey-dry-run.md)을 실행합니다. |
| **4. 출시 전** | 설정, 승인, 모니터링 | [승인](../using/test-approve/gs-approval.md)을 위해 제출하고, 일정과 대상자를 확인하고, [경고](../using/reports/alerts.md)를 활성화합니다. |

**전문가 팁:** 콘텐츠를 구축하기 전에 표현식을 테스트하려면 [개인화 플레이그라운드](../using/personalization/personalize.md#playground)부터 시작하고 과도한 메시지를 방지하기 위해 출시하기 전에 항상 [충돌 감지](../using/conflict-prioritization/conflicts.md)를 확인하세요.

## 주요 용어

Journey Optimizer의 테스트 및 승인 기능을 더 잘 이해하려면 다음의 필수 테스트 개념을 숙지하세요. 각 용어는 자세한 설명서로 연결됩니다.

**[테스트 프로필](../using/content-management/test-profiles.md)** - 개인화된 콘텐츠를 미리 보기 위해 사용되는 가상 고객 프로필(실제 고객이 아님)입니다. 실시간 고객 프로필 서비스에서 플래그가 표시되었습니다. 테스트 모드와 콘텐츠 미리 보기에 필요합니다. [테스트 프로필을 만드는 방법 알아보기](../using/audience/creating-test-profiles.md)

**[테스트 모드](../using/building-journeys/testing-the-journey.md)** - 여정 경로를 통해 테스트 프로필을 전송하는 여정 시뮬레이션 기능입니다. 제한 사항: 초안 여정 전용이고, 네임스페이스가 필요하며, 테스트 프로필만 해당됩니다. [테스트 모드 설명서 보기](../using/building-journeys/testing-the-journey.md)

**[시험 실행](../using/building-journeys/journey-dry-run.md)** - 메시지를 보내거나 API 호출을 수행하지 않고 경로를 추적하는 여정 실행 분석 도구입니다. 사용 사례: 리소스를 사용하지 않고 논리를 검증합니다. [시험 실행에 대해 알아보기](../using/building-journeys/journey-dry-run.md)

**[샘플 입력 데이터](../using/test-approve/simulate-sample-input.md)** - 개인화 테스트를 위한 프로필 속성 값이 포함된 CSV 또는 JSON 파일입니다. 최대 30개의 변형을 지원합니다. 테스트 프로필을 만드는 대신 사용할 수 있습니다. [콘텐츠 베리에이션을 시뮬레이션하는 방법](../using/test-approve/simulate-sample-input.md)

**[시드 목록](../using/configuration/seed-lists.md)** - 내부 이해 관계자의 이메일 주소가 실제 게재(테스트 전송 제외)에 자동으로 포함됩니다. 이메일 채널에만 사용할 수 있습니다. 사용 사례: 품질 모니터링 및 규정 준수에 사용할 수 있습니다. [시드 목록 구성](../using/configuration/seed-lists.md)

**[콘텐츠 실험](../using/content-management/get-started-experiment.md)** - 콘텐츠 베리에이션을 비교하는 A/B 테스트 또는 멀티암드 벤딧 실험입니다. 캠페인만 해당되며, 여정에서는 사용할 수 없습니다. [실험 시작](../using/content-management/get-started-experiment.md) | [실험 만들기](../using/content-management/content-experiment.md)

**[교정쇄](../using/content-management/proofs.md)** - 테스트 프로필 데이터를 사용한 특정 이메일 주소로 전송되는 이메일 게재를 테스트합니다. 시드 목록과 다릅니다(교정쇄는 수동 테스트 전송이며 시드 목록은 자동 이해 당사자 사본입니다). [교정쇄 보내기](../using/content-management/proofs.md)

**[충돌 감지](../using/conflict-prioritization/conflicts.md)** - 동일한 대상자를 타기팅한 겹치는 캠페인과 여정을 식별하는 도구입니다. 여정 지원이 제한되어 있습니다. 단일, 대상자 선별, 대상자 읽기 유형만 지원합니다. [충돌 관리 알아보기](../using/conflict-prioritization/gs-conflict-prioritization.md)

**[승인 워크플로](../using/test-approve/gs-approval.md)** - 활성화 전에 이해 관계자의 승인이 필요한 다단계 검토 프로세스입니다. 승인 정책 구성이 필요합니다. [승인 설정](../using/test-approve/gs-approval.md) | [정책 만들기](../using/test-approve/approval-policies.md)

**[렌더링 테스트](../using/content-management/rendering.md)** - 이메일 클라이언트(Gmail, Outlook, Apple Mail) 및 디바이스 간 이메일 표시 유효성 검사입니다. Litmus 통합이 필요합니다. [이메일 렌더링 테스트](../using/content-management/rendering.md)

**[개인화 플레이그라운드](../using/personalization/personalize.md#playground)** - 샘플 데이터를 사용하여 개인화 구문을 실험하고 표현식을 테스트하는 인터랙티브 학습 환경입니다. 라이브 데이터 세트가 필요하지 않습니다. [플레이그라운드 액세스](../using/personalization/personalize.md#playground)

>[!TAB 종속성 및 관계]

테스트 기능이 서로 연결되고 보다 광범위한 Journey Optimizer 워크플로와 연결되는 방식을 이해합니다. 이 섹션에서는 전제 조건, 업스트림/다운스트림 종속성, 널리 쓰이는 기능 조합을 설명합니다.

+++**전제 조건(테스트 전 필수)**

* 테스트 모드 또는 콘텐츠 미리보기를 사용하기 전에 테스트 프로필을 만들어야 합니다
* 승인을 위해 제출하기 전에 승인 정책을 구성해야 합니다
* 캠페인/여정에 추가하기 전에 시드 목록을 만들어야 합니다
* 이메일 렌더링 테스트에는 Litmus 통합이 필요합니다
* 테스트 모드를 사용하려면 여정이 초안 상태여야 합니다
* 여정에는 테스트 모드를 사용하도록 구성된 네임스페이스가 있어야 합니다

+++

+++**테스트가 종속되는 사항(업스트림)**

* 콘텐츠 제작: 테스트할 캠페인 또는 여정 필요
* 테스트 프로필: 테스트 모드와 콘텐츠 미리 보기에 필요
* 승인 정책: 승인 워크플로에 필요
* 구성: 채널 구성, 이메일 인증, 도메인 설정

+++

+++**테스트에 따라 달라지는 사항(다운스트림)**

* 캠페인/여정 활성화: 오류를 해결하지 않으면 활성화할 수 없음
* 게시: 게시 전에 승인이 필요할 수 있음
* 실시간 모니터링: 출시 후 모니터링 및 보고
* 최적화: 테스트 결과를 사용하여 향후 캠페인 개선

+++

+++**관련 기능**

* 테스트 + 승인 워크플로 - 품질 보증 프로세스
* 테스트 + 충돌 감지 - 고객에게 과다한 메시지 전송 방지
* 테스트 + 콘텐츠 실험 - 성능 최적화
* 테스트 + 보고 - 지속적인 개선 주기
* 테스트 프로필 + 개인화 - 콘텐츠 유효성 검사
* 시험 실행 + 테스트 모드 - 포괄적인 여정 유효성 검사

+++

+++**일반적인 기능 조합**

* 콘텐츠 테스트: 테스트 프로필 + 샘플 입력 데이터 + 개인화 플레이그라운드
* 이메일 유효성 검사: 테스트 렌더링 + 스팸 점수 + 테스트 프로필 + 교정본
* 여정 유효성 검사: 테스트 모드 + 시험 실행 + 테스트 프로필
* 출시 전 체크리스트: 모든 기술 테스트 + 충돌 감지 + 승인 워크플로

+++

>[!TAB 일반적인 질문]

+++**Q: 캠페인을 출시하기 전에 필요한 테스트는 무엇인가요?**

**최소:** 테스트 프로필을 사용한 콘텐츠 미리 보기 + 스팸 점수 확인(이메일)
**권장:** + 이메일 렌더링 + 충돌 감지 + 승인 워크플로
**모범 사례:** + 샘플 입력 데이터 테스트 + 시드 목록 + A/B 실험(최적화하는 경우)

+++

+++**Q: 테스트 프로필을 많이 만들지 않고도 개인화 설정을 테스트하려면 어떻게 해야 하나요?**

**기본 솔루션:** CSV/JSON 파일의 [샘플 입력 데이터](../using/test-approve/simulate-sample-input.md) 사용(최대 30개의 변형 지원)
**대체 솔루션:** 주요 세그먼트를 다루는 3~5개의 대표 [테스트 프로필](../using/audience/creating-test-profiles.md) 만들기
**학습 도구:** [개인화 플레이그라운드](../using/personalization/personalize.md#playground)에서 먼저 실험

+++

+++**Q: 여정의 테스트 모드와 시험 실행 간의 차이점은 무엇인가요?**

**테스트 모드:** 여정을 통해 테스트 프로필을 전송하고, 실제 액션을 트리거하며, 테스트 메시지를 생성합니다. 초안 여정 + 네임스페이스가 필요합니다.
**시험 실행:** 아무것도 보내지 않고 실행 경로를 추적합니다. 모든 여정 상태에서 작동합니다. 보낸 메시지가 없으며 실행한 액션이 없습니다.
**함께 사용:** 메시지 테스트를 위한 테스트 모드 + 논리 유효성 검사를 위한 시험 실행 - 포괄적인 범위.

+++

+++**Q: 프로덕션/실제 운영 상태에서 여정을 테스트할 수 있나요?**

**테스트 모드:** 아니요 - 초안 여정 전용
**시험 실행:** 예 - 모든 여정 상태에서 작동
**콘텐츠 미리 보기:** 예 - 언제든지 개별 메시지 미리 보기
**해결 방법:** 실제 여정을 복제하여 전체 테스트 모드 유효성 검사를 위한 초안 생성

+++

+++**Q: 외부 통합이 필요한 테스트 기능은 무엇인가요?**

**이메일 렌더링:** Litmus 통합 필요(별도 라이선스)
**다른 모든 기능:** Journey Optimizer에 내장되어 있으며 추가 통합이 필요하지 않음
**참고:** 테스트 프로필에는 실시간 고객 프로필 서비스(포함)가 필요합니다.

+++

+++**Q: API 트리거 캠페인을 테스트하려면 어떻게 해야 하나요?**

**옵션 1:** 프로그래밍 테스트에 [캠페인 시뮬레이션 API](https://developer.adobe.com/journey-optimizer-apis/references/simulations/){target-&quot;_blank&quot;} 사용
**옵션 2:** UI에서 테스트 프로필로 콘텐츠 미리 보기
**옵션 3:** 테스트 이메일 주소로 교정본 보내기
**모범 사례:** 포괄적인 유효성 검사를 위해 세 가지 모두 결합
